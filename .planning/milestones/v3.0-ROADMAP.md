# v3.0 x64 Port — Phases 31–36 (SHIPPED 2026-06-20, passed_with_carryforwards)

**Milestone Goal:** Port the modernized client to a native **x64** build, keeping **both** renderers
(D3D9 gl05/06/07 + D3D11 gl11) bootable to character select, to eliminate the 32-bit-bound defects
root-caused across v2.x (the cantina door-snap, the chronic address-space OOM crash) and remove the
x64-hostile legacy D3DX dependency. **Core invariant (extended):** every client-touching phase must
reach boot-to-character-select parity in the **x64** build under both `rasterMajor=5` (D3D9) and `=11`
(D3D11), and must never regress either renderer in the 32-bit build. Reference: SWG Restoration's
stable x64 D3D9 client at `D:\SWG Restoration\x64`; the Miles 9.3b SDK at `D:/Code/milesss-v9.3b/`.

**Outcome:** Definition of done MET and user-verified in-world. 12/13 requirements met; VERIFY-01
(door-snap) PARKED as a pre-existing, bitness/renderer-independent floor-mesh/portal-seam engine quirk
(out-of-scope for the x64 port — see below); VERIFY-03 (CORNERSNAP probe strip) DEFERRED as its
dependent cleanup. Audit: `milestones/v3.0-MILESTONE-AUDIT.md`.

**Dependency chain (why this order):** 64-bit-correctness source work (BITS) and D3DX removal (SHADER)
are prerequisites for a clean x64 build, so they front-load (31–32) before the first linking x64 client
(33). The two renderers are separately-verifiable boot gates (D3D9 in 33, D3D11 in 34). The Miles
7.2e→9.3b port is a self-contained chunk (35). Verification + CORNERSNAP cleanup come last (36).

#### Phase 31: 64-bit Correctness Foundation (BITS-01/02/03) — Complete 2026-06-16
**Goal**: The whole source tree compiles x64-clean — x64-illegal inline asm replaced with intrinsics,
pointer/integer truncation defects fixed, data-layout (struct packing / `sizeof` / serialization width)
audited for IFF/TRE + network-message correctness.
**Result**: 9/9 plans (6 base + 3 gap-closure increments authored mid-execution as the scratch-x64
sweep unmasked successive pre-existing class-(B) layers). x87 FPU asm → `_control87`; SSE math →
intrinsics; pointer/int truncation fixed (C4311/4312/4244 as errors); AutoDeltaMap/PackedMap/Set
uint32_t wire-stable count pin (~753-error C2665/C2668 cascade resolved). 32-bit dual-renderer boot
non-regression certified. Has `31-VERIFICATION.md`.

#### Phase 32: D3DX → d3dcompiler_47 (SHADER-01) — Goal met on x64
**Goal**: Port the legacy D3DX shader-compile path to `D3DCompile` and remove x64-hostile D3DX.
**Result**: SHADER-01 MET ON x64 — D3DX dropped from the x64 link (SwgClient.vcxproj x64 block:
`d3dx/d3dx9/d3dx9dt DROPPED`, SDK supplies x64 d3dcompiler), 0 `D3DXCompileShader/D3DXAssembleShader`
calls in `src/engine/`, both renderers boot x64. Phase-32 plan tracking (1/5) lagged reality — the x64
D3DX removal landed via the Phase 33 X64-01/04 link work + the asm-path D3DAssemble port (33-02), not
Phase-32 plan check-offs. 32-01 (D3DAssemble census GATE) complete; 32-02..05 superseded by the x64-era
path. The 32-bit Fix-A SEH guard retained.

#### Phase 36: Verification & CORNERSNAP Cleanup (VERIFY-01/02/03) — Concluded 2026-06-20
**Goal**: Confirm the door-snap + OOM-crash classes resolved by x64, then strip the CORNERSNAP probes.
**Result**: 1/2 plans (partial by design).
- **VERIFY-02 PASS** — extended x64 session, no `MemoryManager` OOM (`b0780503` class); decelerating
  memory floor, flat handles (baseline 1819→1858 MB over door cycles).
- **VERIFY-01 FAIL → PARKED** — the cantina door-snap persists in the x64 build (gl05/D3D9),
  falsifying the long-held "x64/deterministic-SSE fixes it" hypothesis. Root-caused via live
  non-debugger DBWIN capture + a 4-AI crew as a **pre-existing, bitness/renderer-INDEPENDENT engine
  quirk**: the floor-collision walk reacts to the cantina doorway FLOOR-MESH SEAM (footprint circle
  clips an uncrossable seam edge → `PWR_HitEdge` → `RR_Resolved` rewind), not any wall. Orthogonal to
  the x64 port; reclassified out-of-scope-for-v3.0 and carried forward. Evidence: `36-01-SUMMARY.md`,
  `research/CONSULT-36-doorsnap-SYNTHESIS.md`; reusable `tools/setup/dbwin-cornersnap-capture.ps1`.
- **VERIFY-03 / plan 36-02 DEFERRED** — strip CORNERSNAP probes; gated on VERIFY-01 sign-off (never
  given). Probes KEPT as the door-snap acceptance harness for the future fix.
