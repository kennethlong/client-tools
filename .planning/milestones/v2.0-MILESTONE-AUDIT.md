# v2.0 "Modernisation" — Milestone Audit

**Audited:** 2026-05-25 · **Status:** `tech_debt` (functional PASS, artifact integrity degraded — accepted ad-hoc drift)

## Bottom line

The milestone **shipped its core value**: a modern MSVC-STL build that boots to character select and renders a ground scene under *both* a D3D9 and a new D3D11 renderer, selectable by `rasterMajor`. End-to-end flows pass empirically (verified by direct testing 2026-05-25) and by codebase wiring (single `Gl_api` loader in `Graphics.cpp:185-253` multiplexes both renderer DLL families; STL swap leaves no `_STLP_*`/stlport conflict surface; DPVS Option-α edits are live).

There are **zero functional cross-phase wiring breaks.** The audit's one substantive finding is **artifact integrity**, not function: several Phase 7/9 requirements are documented against a CMake tree the project abandoned (Phase 9 Option-D base swap to Koogie's MSBuild tree). This is accepted ad-hoc drift, not a blocker.

## Integration & flows

- **Renderer coexistence (P11↔engine):** WIRED. `Graphics.cpp` selects gl05/06/07 (D3D9) vs gl11 (D3D11) by `rasterMajor`; same Gl_api table bound; D-05 (D3D9 stays buildable) holds.
- **STL coherence (P9↔all):** WIRED. `unordered_map` aliasing live; no FORCE:MULTIPLE/stlport/_STLP_*.
- **DPVS (P10↔engine):** WIRED. OCCLUSION_CULLING bit stripped in RenderWorld.cpp (debug+release); indoor portals retained; throwaway instrumentation removed.
- **E2E:** build → boot → char-select → zone-in → render PASSES under both rasterMajor=5 and =11.

## Recommendation

**Proceed with the v2.0 close, accepting the documented divergence.** The CLEAN-01..04 "dead code still present in the active tree" is real residual scope but is (a) non-breaking and (b) a future decision (re-do removal against the Koogie tree only if still desired) — not a v2.0 gate. Do **not** insert GSD closure phases for documentation drift. Carry the tech-debt items forward; the genuine forward work (asset PS pipeline etc.) is the next milestone's Phase 12, not v2.0 remediation.
