// Structure definition:
struct OGG_INFO
{
    U8* Ogg_file_image;
    S32 Ogg_image_size;
    U8* start_Ogg_data;
    U8* end_Ogg_data;
    S32 page_num;
    S32 sample_count;
    S32 byte_offset;
    S32 next_page_expected;
};
Description

Members
Ogg_file_image [in]

Original Ogg_file_image pointer passed to AIL_inspect_Ogg.
Ogg_image_size [in]

Original Ogg_image_size passed to AIL_inspect_Ogg.
start_Ogg_data [in]

Pointer to start of data area in file.
end_Ogg_data [in]

Pointer to last valid byte in data area.
page_num [in]

32-bit page sequence number from the Ogg header at byte_offset.
sample_count [in]

Total number of samples already generated by encoder at the time the current page was written.
byte_offset [in]

Offset of page from start_Ogg_data (i.e., suitable for use as loop point).
next_page_expected [in]

Offset from start_Ogg_data where the next page is expected. If the next call to AIL_enumerate_Ogg_pages fails to find a valid OggS identifier at this offset, it will assume the file image is corrupt and return 0 to stop enumeration.
Discussion

All OGG_INFO fields other than those documented above are for internal use, and may change between MSS versions.

Fields between Ogg_file_image and end_Ogg_data, inclusive, are valid as soon as AIL_inspect_Ogg returns, even if the file contains no valid Ogg pages.

All remaining fields pertain to the current page being inspected, and are valid only after AIL_enumerate_Ogg_pages has returned TRUE.

Ogg page numbers normally begin at zero and increment by one at each page boundary until end-of-file is reached.
