// Function prototype:

S32 AIL_filter_property(HPROVIDER filter,
char const * name,
void* before_value,
void const * new_value,
void* after_value);
Description

retrieves and/or sets the current value of the specified provider-level property.

Parameters
filter [in]

specifies the filter to query.
name [in]

specifies the name of the property to retrieve.
before_value [out]

is a pointer to a variable that will contain the property value before changing it. Always use this pointer to retrieve read-only properties. You can use 0 if you don't want to retrieve the prior value.
new_value [in]

is a pointer to a new value to set. If you don't want to set the property, pass in 0.
after_value [out]

is a pointer to a variable that will contain the property value after changing it. You can use 0 if you don't want to retrieve the post-value.
Return value
return [out]

Non-zero for success, 0 for failure.

Discussion

Filter-level properties are properties that are read globally from a filter. For properties that are specific to each sample (sample-level properties), use the AIL_sample_stage_property function.

This function can retrieve or set the current value of a property.

Example files that use this function: filter.cpp, multiflt.cpp
