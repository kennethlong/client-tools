Overview

Busses allow you to specify a group of samples to mix together and apply effects and attenuation to the group.

Initially busses are disabled - all samples are mixed together and then written to the hardware. Under the Busses section of the Settings you can Activate busses, which will create an aux bus you can mix to.

As with all sound matching in Miles Studio, the sounds to be mixed on an aux bus is determined by a label query. If a sound does not match any aux bus, then it is mixed on the main bus. This means that an empty label query on an aux bus will prevent any sounds from mixing on the main bus.

Using Busses

The primary reason for using a bus is to use of the signal strength of the mixed samples as an input to something else.

There are two ways of doing this:

  1. A side chain compressor can be installed on a bus via Preset. To do so, after activating busses, create a preset and add a filter stage. Select the input from the drop down and then set the preset on the bus you want to duck in the Busses selection.

  2. The signal strength of each active bus is routed to a variable: "_MainBusDb" for the main bus, and "_AuxBusDb#", where # is the index of the bus. The value will be between -96 and 0. In addition, a similar pair of variables "_MainBusDAC" and "_AuxBusDAC#" are set that will be between 0 and 1. In order to use these, create the variable matching the name and they will be automatically set.

In addition, a limiter can be applied to each bus independently. Currently this limiter is not configurable, but just makes an effort to prevent clipping.
