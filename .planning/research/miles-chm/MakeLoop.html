The MAKELOOP.EXE utility is a Windows console application that creates perfectly looped files in Bink Audio, MP3, or ADPCM format. For MP3, it works with the LAME MP3 encoder, and for Bink it uses the BinkA_encode tool.

MakeLoop will accept PCM .WAV, ADPCM .WAV, or existing .MP3 files as input. MakeLoop resamples your input file to the next-smaller multiple of the frame size (usually 1152 samples for MP3, for example), then launches the appropriate encoder with a combination of options chosen for loop-transition quality. Finally, MakeLoop post-processes MP3 files to remove silent frames.

MakeLoop is convenient because you don't need to specify subblock offsets with AIL_set_sample_loop_block or AIL_set_stream_loop_block to loop a file encoded by it. Just use AIL_set_sample_loop_count or AIL_set_stream_loop_count to loop the entire file. Most loopable .WAV files will sound great when encoded with MAKELOOP.EXE, saving your production department the need to select and audition loop points manually with the Miles Sound Studio.

MakeLoop works by prepending a copy of the last frame in the file to the beginning of the file. This makes the file appear to be a continuous stream from the encoder's point of view, with the goal of preventing mismatches between Bink Audio's overlaping blocks, ADPCM block initializers and MP3 overlap-window contents when the file is looped at runtime.

Then the file is then compressed with either the LAME command-line encoder, the BinkA_encode command-line tool, or the with our ADPCM compression function, depending on the specified file type. Finally, we post-process the encoded output file to remove unnecessary frames, including silent and metadata frames added by LAME and the copy of the final frame that was prepended earlier.

You can run MAKELOOP.EXE with no command-line options to see usage notes for the current version. Most .WAV files can be encoded simply by running makeloop infile.wav outfile.mp3 to generate an .MP3 output file, or makeloop infile.wav outfile.binka to generate an Bink Audio output file or makeloop infile.wav outfile.wav to generate an ADPCM .WAV file.
