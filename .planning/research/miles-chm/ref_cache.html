Overview

Cache and Purge Sounds load and unload sounds from memory, for use in Start Sound events.

  1. The bank to load or unload from. Only one soundbank's sounds can be affected in a single cache/purge step.
  2. The number of bytes this will load. With respect to Asset Specialization, this uses the assets for the current platform and language for computing the size.
  3. The sound list to load. Entries that do not exist will be highlighted in red. Sounds can be dragged to this pane to add them.
  4. Button to add sounds. This is the only way to add more than one sound at once.
  5. Clear any bad (red) references to sounds that do not exist.

It's important to note that the sounds will take a while to load, and it's generaly unpredictable which sound will be loaded first in the list. If you have a situation where you need to load a lot of sounds, but one in particular needs to be loaded first (say, for main menu music), you should make two cache sound event steps, and load the urgent sounds first.
