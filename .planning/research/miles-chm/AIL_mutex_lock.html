// Function prototype:

void AIL_mutex_lock();
Description

suspends the Miles background thread by locking the background mutex.

Return value
return [out]

Discussion

Calls to AIL_mutex_lock and AIL_mutex_unlock should bracket any sections of application code which must not be interrupted by the background Miles thread.

This might block for a period of time, waiting for the lock to be taken. A lower-overhead lock can be taken with AIL_lock, which doesn't wait until the background Miles thread is inactive, but instead will only block it, if is already inactive.
