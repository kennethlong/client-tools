// Function prototype:

S32 AIL_enumerate_filter_sample_properties(HPROVIDER filter,
HMSSENUM* next,
RIB_INTERFACE_ENTRY* dest);
Description

enumerates the possible sample-level filter properties.

Parameters
filter [in]

specifies the filter for which to enumerate the sample-level properties.
next [out]

is a pointer that MSS uses to keep track of the next property to return. You initialize the next variable with the constant MSS_FIRST before the first call to AIL_enumerate_filter_properties.
dest [out]

points to an structure of type RIB_INTERFACE_ENTRY that receives the property information. The most important field in this RIB structure is "entry_name" which is a pointer to the property name (which can then be used in the AIL_sample_stage_property function.
Return value
return [out]

0 if no further sample properties were found; non-zero if another property was found.

Discussion

Sample-level properties are properties that are set for a single sample. For properties that are global to the filter, use the AIL_enumerate_filter_properties function.

Example files that use this function: filter.cpp, multiflt.cpp
