Using the Event System

Basic integration with the Miles high level event system is incredibly simple. It's designed around a "Fire and forget" system, where you enqueue events (preferably by name), and once a frame you tick the event system, durign which it executes the events. Here's a quick example of how to get the event system up and running: 

    // Somewhere in the file add #include "mss.h"

    // Initialize Miles
    AIL_startup();
    
    // Create a driver to render our audio - 44khz, 16 bit,
    HDIGDRIVER dig = AIL_open_digital_driver(44100, 16, 2, 0);

    // Create an event system tied to that driver - let Miles choose memory defaults.
    AIL_startup_event_system(dig, 0, 0, 0);

    // Wow, that's really it?

The above code is all the initialization code required to get the system up and running. However, the system still needs to tick. Wherever you process your frame, add this code: 

    AIL_begin_event_queue_processing();

    AIL_complete_event_queue_processing();

We're up to 5 function calls and the system is up and running. It's time to start playing with events. To do this, we call AIL_enqueue_event_by_name(). 

    void Actor_Player::FireMyGuns(S32 i_ReallyFast)
    {
        // Spawn a bullet!
        
        // Use up ammo!

        // And play a sound from the "Effects" sound bank, in the "Player" folder, called "Fire".
        AIL_enqueue_event_by_name("Effects/Player/Fire");
    }

At this point, we've got the system up and running, it's ticking, and we have an event that will try to play. But where will it get that data?

We need to load a soundbank!

Back up after the startup code, add this: 

    AIL_add_soundbank("Effects.msscmp", "Effects");

And that's all it takes for basic usage. However, you'll want to integrate the audition system as well.

It's significantly more complicated.

Using the Audition System

    // Do this after AIL_startup_event_system() and *before* AIL_add_soundbank()

    // Initialize the audition system. Use server assets, default platform, and default language
    AIL_audition_startup(0, 0, 0);

    // Connect to the current machine for Miles Auditioning.
    AIL_audition_connect(AIL_audition_local_host());

I lied, sorry. It's not complicated - that's it!

A quick footnote - the audition system is implemented in a separate library, so that when you release you don't have a bunch of code lying around not getting used. However, that means you'll need a way for ship to not call these functions, otherwise you'll have a dependency around you don't want. For example: 

#ifndef SHIP_GAME
    AIL_audition_startup(0, 0, 0);
    AIL_audition_connect(AIL_audition_local_host());
#endif

For basic integration of Miles, you're done. For more advanced integration techniques, check out Advanced Topics. If you've got all you need from the code side of things, head over to Getting It In The Game.
