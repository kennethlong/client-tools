Miles Studio 9.1 introduces Settings.

Settings are global definitions referenced by all of your banks. They are designed to be the one place to store names, labels, and variables in your game. A variety of changes are associated with this and are described in Settings.

If you only use one bank in your game, this will be easy. Simply open your game's bank, and the upgrade process will guide you. All assets will be affected in one way or another, so expect a large source control changelist.

If you have multiple banks, you'll need to designate one to contain your settings. You'll have to upgrade this bank first. Then once you have this upgraded and loaded in the tool, load and upgrade your other banks. An important thing to note is whichever bank you chose must be loaded first in the game! It is completely valid to have a bank which solely contains your settings. In this case, create the new bank and save it, then load and upgrade all of your other banks.

In either case, the upgrade process will look through the bank you load, and automatically create settings based on the loaded bank, migrating the definitions to a settings asset. In addition, the compiled banks must be rebuilt for the runtime upgrade.

Other new features:

  • Event Coloring! By request, you can assign a color to your events to easily group them visually in the asset list.

  • Implicit State Variables. In a Start Sound, that ugly edit box with the huge string in it is gone.

  • Blends are easier to set up. Define the Blend in your settings, and select it in a Start Sound. Easy peasy!

  • Cache/Purge sounds is easier to use. Also it's easier to spot dangling references.

  • Dragging to assign labels! Drag an event to a label under the Settings/Labels tree node and it will assign that label to all Start Sounds in the event.

  • Bank properties are a little more visible. Granted right now the only property is whether it's under Source Control or not, but there are some other useful things in there.

  • Setting up Limits is a lot more straightforward.

  • Loop point editing is more intuitive and less programmer-y.
