This guide will demonstrate how to categorically adjust the volume of sounds based on game input. This will introduce hooking Ramps up to Variables.

Requisite Terminology

  1. Label
  2. Label Query
  3. Ramp
  4. Variable

New Terminology
None!

Steps

  1. Create two labels, name them "Effect" and "Music".



    These labels are the two groups of sounds that will be affected by the in-game volume control. For the purposes of the example, sounds that are started should either have the label Effect, or Music, but not both. This allows Miles to distinguish between them.

  2. Create two variables named UserEffectVol and UserMusicVol



    These variables are assumed to be set by the game. They should be assigned to the decibel volume desired for the sounds with the labels Effect and Music, respectively.

  3. Create two ramps named "EffectRamp" and "MusicRamp".



    These ramps are going to do the actual work - they will update the actual volumes for the desired sounds, but first they must be set up.

  4. Select the EffectRamp, and add the label "Effect" to the Labels To Affect. Then click "Assign Variable" and select "UserEffectVol".



  5. Select the MusicRamp, and add the label "Music" to the Labels To Affect. Then click "Assign Variable" and select "UserMusicVol".



    Since ramps, by default, affect all sounds, we need to adjust the label query for each ramp such that they only affect the desired sounds. For the EffectRamp, we want it to only affect sounds with the Effect label, so we set the label query to that. Similarly for MusicRamp.

    In order to tie the volumes to the variables the game sets, we assign the Target Value for the ramp to the respective variable using the "Assign Variable" button.

At this point, everything is ready. We don't ever need to use an Apply Ramp event step, because the ramp will "chase" the variable's value at all times. Note that this means that the volume will move as fast as the variable changes!
