// Function prototype:

S32 AIL_enumerate_filters(HMSSENUM* next,
HPROVIDER* dest,
char** name);
Description

enumerates the available filter processors for display or for opening.

Parameters
next [out]

is a pointer that MSS uses to keep track of the next provider to return. You initialize the next variable with the constant MSS_FIRST before the first call to AIL_enumerate_filters.
dest [out]

points to an address that receives the HPROVIDER handle that is used for AIL_open_filter.
name [out]

receives a pointer to the provider's text description (such as "High-pass filter"). This parameter is usually used to match the filter you're interested in.
Return value
return [out]

0 if no further filters were found; non-zero if another filter was found.

Discussion

You can call this function before AIL_open_filter or AIL_set_sample_processor to find out what filters are installed on the system. If you know the specific filter than you want to use, then you probably want to use the AIL_find_filter function.

The HPROVIDER handle values returned by this function can change from one program execution to the next. For example, the high-pass filter may have the HPROVIDER handle value "1" during one execution, and the value "2" the next. You must, therefore, always use these functions to obtain the proper HPROVIDER handle value at runtime.

Output filters such as Dolby, DTS, and SRS matrix filters and the DirectSound3D/EAX voice filters have an .FLT extension, but they won't be enumerated by this function. This function is used only to obtain a list of available per-sample DSP pipeline filters.

Example files that use this function: FILTER.CPP
