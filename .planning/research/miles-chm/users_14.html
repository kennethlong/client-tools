This guide will cover how to set up busses such that music is automatically ducked by a dialog bus.

Requisite Terminology

  1. Label & Label Query
  2. Preset
  3. Events & Sounds

New Terminology

  1. Bus: A mixed group of samples, set up such that you can modify them as you would any other sample.

Steps

  1. Create a label and name it "Dialog".

  2. Load a music track, and create an event to play it.

  3. Load a dialog track, and create an event to play it. Assign it the Dialog label.


  4. Select the "Busses" setting, and click "Activate".


    [after clicking]


    This tells Miles to initialize the bus system and create two busses. The first is the Main bus, which all samples are assigned to by default. However, with the addition of a new bus, all samples matching the label query for the bus are assigned to it. Since the default label query is empty, that means all samples will go to that bus, so we narrow the query to only assigning sounds with "Dialog" to the aux bus.

  5. Add the label "Dialog" to "Aux Bus 1".


  6. Create a new Preset, and select the "Compressor" Filter in Stage 1.

  7. Select "Aux Bus 1" for the "Input" Parameter of the Compressor.


    The compressor filter will attentuate based on the strength of an input signal. In this case, we want to attentuate the Main bus, where our music will be playing, based on the volume of the Dialog bus, which is on Aux Bus 1.

  8. Select the "Busses" setting, and drag the Preset to the Preset field of the Main Bus.


  9. Audition your music event, then audition the dialog event.
