Description

retrieves and/or sets driver-level properties for an output filter.

Parameters
dig [in]

indicates the driver whose installed output filter should be queried.
property [in]

is a string variable describing the property to retrieve.
before_value [out]

is a pointer to a variable that will contain the property value before changing it. Always use this pointer to retrieve read-only properties. You can use 0 if you don't want to retrieve the prior value.
new_value [in]

is a pointer to a new value to set. If you don't want to set the property, pass in 0.
after_value [out]

is a pointer to a variable that will contain the property value after changing it. You can use 0 if you don't want to retrieve the post-value.
Return value
return [out]

Non-zero for success, 0 for failure.

Discussion

Certain digital sound configurations, including DirectSound3D, EAX, Dolby Surround, DTS, and SRS Circle Surround(r), are implemented with internal extension modules called output filters. Output filters can offer additional control options beyond those available from the core MSS API. Most applications will never need to work with output filter controls directly, but those that wish to do so can obtain a handle to the current voice or matrix output filter from AIL_digital_output_filter and access provider- and driver-level properties via the AIL_filter_property and AIL_output_filter_driver_property functions.

You can find a list of configuration-specific properties in the chapter on the specific output filter you are using. See the examms demo program for an example of how to enumerate and query provider-level, driver-level, and sample-level properties.

Each per-voice output filter, such as those installed when MSS_MC_DIRECTSOUND3D or one of the EAX options is specified in a call to AIL_open_digital_driver, publishes a property called "Maximum supported samples" that returns the total number of 3D samples that can be (potentially) opened by that provider.

Many output filters also publish provider-specific properties -- see the provider information at the end of this chapter for details.

Retrieving Example: To obtain the total number of 3D samples that can be opened, you could use the following code: 

U32 max_3D_samples;
AIL_output_filter_driver_property( opened_dig_driver,
                                   "Maximum supported samples",
                                   &max_3D_samples, 0, 0 );

Setting Example: To set the EAX room style, you could use the following code (you'd usually want to use the AIL_set_room_type function, though): 

U32 room_style=EAX_ENVIRONMENT_BATHROOM;
AIL_output_filter_driver_property( eax_dig_driver,
                                  "EAX environment selection",
                                   0, &room_style, 0 );

(In reality, you still need to check the returned values from AIL_allocate_sample_handle, because not all of those hardware voices may be available by the time they're needed.)

This function will have no effect if no per-voice or matrix output filter is in use.
