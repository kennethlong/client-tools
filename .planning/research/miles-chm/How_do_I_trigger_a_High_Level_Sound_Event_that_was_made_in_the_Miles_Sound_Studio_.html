Q: How do I trigger a High-Level Sound Event that was made in the Miles Sound Studio?

A: To trigger an event, follow these steps:

  1. Initialize and start up the sound system (see this FAQ).
  2. Startup the high-level event system with AIL_startup_event_system.
  3. Open and add the soundbank that contains the event with AIL_add_soundbank.
  4. Queue the event to be triggered by name with AIL_enqueue_event_by_name.
  5. At this point, you can queue as many events as you want...
  6. Once per frame (usually at the end of the frame), you must process the events. You start the processing with AIL_begin_event_queue_processing.
  7. At this point, you can enumerate the sounds that were triggered and associate them with your data, if necessary.
  8. Finally, complete the processing with AIL_complete_event_queue_processing. Your event queue is now empty again, ready for the next frame.

All games that use the high-level API will be structured this way. First, you queue up all your event triggers, and then once a frame you process them all (where they are triggered)!
