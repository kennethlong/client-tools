This walkthrough will discuss decreasing the volume of a group of sounds, so that another may be heard more easily when it plays, and then returning the previous sounds to their original volume upon completion.

Requisite Terminology

  1. Event Asset
  2. Sound Asset
  3. Start Sound Event Step
  4. Control Sounds Event Step
  5. Audition
  6. Labels
  7. Label Query

New Terminology

  1. Ramp: A Ramp is a construct that allows you to interpolate various aspects of a sound over time. They allow for interpolation of volume, wet level, low pass cutoff, and playback rate (pitch). Similar to Control Sounds, they use a label query to determine what sounds to affect. A ramp starts at an initial value, which is the value applied to all matching sounds if the ramp is never altered via any event. The effect a ramp has on a sound is modulated, not replaced - meaning if a volume ramp sets the volume attentuation to -10 dB, and the sound's 3D position causes an attentuation of -10 dB, the resulting volume will be at -20 dB. This holds true for other ramp types as well.

  2. Apply Ramp Event Step: This event step sets the destination for a Ramp, and how long it takes to get there. For example, it may set the target of a volume ramp to -96 dB over a time of 1 second. After 1 second, all sounds affected by the ramp will be inaudible. However, the ramp will start its interpolation at whatever value it was last left at, which might not be at any target value if the interpolation hasn't had time to complete. In more detail, say we have a volume ramp that has been created, and is at the default initial value of 0 dB. All the sounds that are affected by that ramp are not attentuated at all as a result of this ramp. Then we run an Apply Ramp using that ramp, moving it to a target value of -96 dB over 4 seconds. If that were the only event run, in 4 seconds the affected sounds would be inaudible. However, 2 seconds later we run another Apply Ramp using the same ramp, moving it to a target value of 0 dB over 4 additional seconds. The starting position of the second Apply Ramp will be -10 dB instead of -96 dB, since it only had half the time ( 2 seconds instead of 4 seconds ) to ramp the volume, and as a result was at a volume attentuation of 50% when the new Apply Ramp began. However, this only means the *rate* of volume change will be different - it will still take 4 seconds to reach 0 dB.

  3. Completion Event: A completion event is an event that has been set to automatically run when a sound finishes playing.

Steps

  1. Drag two sounds in to Miles Studio and for each of them right click and select "Create Event For Sound". One will be treated as Music for the purposes of this guide, and such should be significantly longer than the other. You should end up with something like this:



  2. Create a new label called "Music" and assign it to whichever sound is longer by dragging the event that starts it to the label's entry in the Asset List.



  3. In the Asset List, Select "Ramps" and create a new ramp called "MusicDuck".



    This creates a Ramp that can be referenced by an Apply Ramp Event Step in an Event. This ramp will be responsible for attentuating the sound with the Music label while the shorter sound plays.

  4. Select the new ramp and click the plus next to "Labels to Control" and pick "Music".



    By default, ramps have an empty label query, so they affect all sounds. We want this ramp to only affect our Music sound. Also note that the Initial Value defaults to "has no effect". For volume ramps, this means 0 dB - they will not attentuate any affected sounds. Similarly, playback rate ramps default to 0 sT, and low pass cutoff ramps default to 24khz.

  5. Now we need to set up the shorter sound to "duck" the longer sound using the new ramp. Select the shorter sound's event, scroll down, and click the "Add New Step" button. Change the new step to "Apply Ramp" via the drop down, and adjust the "Target Value" slider to "-96 dB"



    Now, immediately after the sound is started, the MusicDuck ramp smoothly moves the volume of the sound with the Music label to silence. However, the ramp takes 1 full second, and our sound starts immediately.

  6. Scroll back up in the same event and set the min and max Delay of the start sound event step to 1000 ms.



    This causes the sound to delay starting for 1 second, so the MusicDuck ramp has time to attentuate the sound with the Music label. However the sound with the Music label has no way to return to full volume. To move the sound with the Music label back to full volume, we need another Apply Ramp event step.

  7. Create a new empty event named "NewEvent", add a new event step and change the type to "Apply Ramp". Leave the "Target Value" slider to "0 dB".



  8. Select the shorter sound's event, and drag "NewEvent" to the "Completion Event" field.



    Completion events are events that are automatically run when a sound completes. In this case, the completion event runs the event containing the Apply Ramp event step that returns the sound with the Music label back to full volume.

  9. Audition the event that starts the sound with the Music label, then while that sound is playing audition the event that starts the other sound.

    Due to the effects of the MusicDuck ramp, the second sound played will silence the sound with the Music label before playing. Then, upon completion, it will restore the sound with the Music label to full volume.
