For sounds that don't move, you can attach the position when you enqueue the event: 

	S32 token = AIL_enqueue_event_start();
	AIL_enqueue_event_position(&token, x, y, z);
	AIL_enqueue_event_end_named(token, "Bank/EventName");

If the sound moves, you need to track the sound over time and set the position each frame. In order to do this, you'll need to tie sounds created by the events with your own game's data. There's are two options:

  1. Track the event ID in your game data and when enumerating the active sounds, set the sound's position: 

        AIL_begin_event_queue_processing();

    HMSSENUM Token = MSS_FIRST;
    MILESEVENTSOUNDINFO Info;
    while (AIL_enumerate_sound_instances(0, &Token, 0, 0, 0, &Info))
    {
    	GameData* Data = GameDataLookup(Info.QueuedID);
    	if (Data)
    	{
            Vector3* Position = (Vector3*)Data->Position;
            AIL_set_sample_3D_position(Info.Sample, Position->x, Position->y, Positions->z);
        }
    }

    AIL_complete_event_queue_processing();

  2. Enqueue a reference to your game data in the user buffer. 

    	S32 token = AIL_enqueue_event_start();
	AIL_enqueue_event_buffer(&token, PtrToGameData, 0, 1);
	AIL_enqueue_event_end_named(token, "Bank/EventName");


        AIL_begin_event_queue_processing();

    HMSSENUM Token = MSS_FIRST;
    MILESEVENTSOUNDINFO Info;
    while (AIL_enumerate_sound_instances(0, &Token, 0, 0, 0, &Info))
    {
    	GameData* PtrToGameData = (GameData*)Info.UserBuffer;
        Vector3* Position = (Vector3*)PtrToGameData->Position;
        AIL_set_sample_3D_position(Info.Sample, Position->x, Position->y, Positions->z);
    }

    AIL_complete_event_queue_processing();
