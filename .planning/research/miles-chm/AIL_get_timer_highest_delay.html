// Function prototype:

U32 AIL_get_timer_highest_delay();
Description

returns the highest delay in timer servicing under Win 32.

Return value
return [out]

The highest delay since the last call to this function.

Discussion

This function allows you to measure just how well the OS is servicing the Miles background thread.

You can use the DIG_DS_MIX_FRAGMENT_CNT preference to increase the digital subsystem's mix ahead time to smooth over the delay returned by this function.

The Windows and MacOS thread schedulers tend to have the most trouble during disk I/O.
