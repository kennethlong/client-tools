To build an application that uses Miles, you must include the Miles SDK in the build process. There are four simple steps to do this:

  1. Include MSS.H in your source files.
  2. Link with the MSSXenon.LIB or MSSXenon.LTCG.LIB libraries. Using the LTCG library will result in slightly better performance and memory use, but it takes longer to link, and Edit and Continue will no longer work.
  3. If you use MIDI, link with the MSSXenonMIDI.LIB or MSSXenonMIDI.LTCG.lib library.
  4. If you are using MP3 data, link with the MP3xenon.LIB or MP3xenon.LTCG.LIB libraries. If you are using Bink Audio data, link with the BinkAXenon.LIB or BinkAXenon.LTCG.LIB libraries. If you are using Ogg Vorbis data, link with the Oggxenon.LIB or Oggxenon.LTCG.LIB libraries. If you are using XMA data, link with the XMAxenon.LIB or XMAXenon.LTCG.LIB libraries. Don't forget the Register_RIB calls needed to enable these libraries.
  5. If you are the software DSP filters, link with the fltxenon.LIB or fltxenon.LTCG.LIB libraries.

And that's it - you should now be able to call the Miles functions!
