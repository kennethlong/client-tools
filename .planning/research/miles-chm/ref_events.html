Events are the tools sound designers use to play, alter, and otherwise affect the soundscape of the game. They consist of a sequence of Event Steps. Each event step occurs in order, doing exactly what you tell it to. Many step types exist:

  • Start Sound - Used to start one or more sounds.
  • Control Sounds - Used to manipulate running sounds.
  • Apply Environment - Alter the way all sounds are heard.
  • Comment - Used to document. Ignored by the runtime.
  • Cache Sounds - Load sound data for use by Start Sound.
  • Purge Sounds - Free sound data previously loaded.
  • Set Limits - Set how many sounds can play for a label.
  • Persist Preset - Set a preset to affect current or future sounds.
  • Add Ramp - Ramp aspects of current or future sounds.
  • Set Blend - Set up how engine sounds crossfade and pitchbend.
  • Execute Event - Queue another event to run.
