// Function prototype:

void AIL_set_sample_speaker_scale_factors(HSAMPLE S,
MSS_SPEAKER const * dest_speaker_indexes,
F32* levels,
S32 n_levels);
Description

sets the speaker scale factors for a sample.

Parameters
S [in]

indicates the HSAMPLE for which the speaker scale factors are to be set.
dest_speaker_indexes [in]

specifies an array of one or more speakers.
levels [in]

specifies an array of coefficients from 0.0 to 1.0 which will scale the volume response at each specified output channel, from 0.0 (silence) to 1.0 (full response).
n_levels [in]

specifies the number of entries in dest_speaker_indexes, and levels.
Return value
return [out]

Discussion

When the Miles software mixer is in use, this function may be called to suppress or amplify the final sample's output on a per-speaker basis. Unlike AIL_set_sample_channel_levels, it is applied on both 3D and 2D samples. For instance, you could turn off certain speakers completely. By default, all speaker response scalars are set to 1.0 when the driver is opened.

The levels values are linear scalars, where each doubling of the level value represents a 6-dB change in the wet or dry level for that channel. For instance, a value of 0.5 is 6 dB quieter than a value of 1.0, and 0.25 would be 12 dB quieter than 1.0.

For a detailed understanding of channel-based control functions such as this one, you should carefully review the How do I use multi-channel sound output in Miles? FAQ.
