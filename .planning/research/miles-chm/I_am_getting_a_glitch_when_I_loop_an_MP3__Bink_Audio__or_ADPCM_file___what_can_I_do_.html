Q: I am getting a glitch when I loop an MP3, Bink Audio, or ADPCM file - what can I do?

A: Short answer: Use the Miles Sound Studio to encode any .MP3, Bink Audio, or ADPCM .WAV files that need to be looped at runtime. The Miles Sound Studio can process these files automatically for seamless or near-seamless looping. You can also place loop markers by hand in files that still exhibit glitches after automatic loop optimization.

You can also make loopable sound files using our command line MakeLoop tool. MakeLoop uses the same techniques as the Miles Sound Studio, but because it's a console application, it's suitable for batch processing large numbers of files at once. It may also be easier to integrate into legacy production workflows that don't include the Miles Sound Studio.

Long answer: A number of things can go wrong when trying to loop or seek within a block-oriented file or stream. For example, MP3 decoding is a DSP-intensive process that uses discrete mathematical transforms to operate on continuous real-world data -- which is itself partitioned into discrete frames! What that means is that MP3 frames can't be interpreted in a vacuum. To decode a given frame properly, the decoder must overlap its data with data left over from the previous frame. Worse, using an entirely-unrelated feature called the bit reservoir, MP3 files can 'borrow' space needed to store data for a particular frame from several of the preceding frame. These two factors make it unlikely that an arbitrary attempt to seek from one part of an MP3 file to another will be free of objectionable artifacts.

ADPCM files aren't as complicated, but they also use a block-oriented structure with similar implications. In either case, MAKELOOP.EXE and the Miles Sound Studio work by decimating your input file, or resampling it to conform to the next-smaller multiple of the encoded frame size. Frame sizes are typically very small -- on the order of 1000-4000 samples for ADPCM files and 1152 samples for MP3 files -- so this resampling process does not usually cause a large change in the pitch of the encoded sample.

One other note: if you're encoding .MP3 files yourself with the popular open-source LAME (Lame Ain't an MP3 Encoder) package, we strongly recommend using the --nores option to encode any files that will undergo looping or seeking. MakeLoop and the Miles Sound Studio will handle this and other details for you.
