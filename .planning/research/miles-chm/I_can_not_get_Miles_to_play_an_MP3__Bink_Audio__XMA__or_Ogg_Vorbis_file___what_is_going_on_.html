Q: I can not get Miles to play an MP3, Bink Audio, XMA, or Ogg Vorbis file - what is going on?

A: If you can't play an MP3, Bink Audio, Ogg Vorbis, or similar file, try these troubleshooting questions:

  1. Did you copy all the redistribution files into a directory available to your application? Are they up to date with the version of Miles that you're using?
  2. On Windows, MacOS and GNU/Linux, did you call AIL_set_redist_directory to tell Miles where this directory is? This directory is relative to the location of the Miles library file, so if you are using a relative path - make sure it is relative to the right directory!
  3. On platforms that use static libraries, did you call Register_RIB(MP3Dec) to tell Miles to load the MP3 decoder, Register_RIB(OggDec) to load the Ogg Vorbis decoder, Register_RIB(BinkADec) to load the Bink decoder, or Register_RIB(XMADec) to load the Xbox 360 XMA decoder?
  4. Are you using AIL_set_named_sample_file to tell Miles what type of data it is?
  5. Did you sign the license agreements for MP3 and/or Ogg Vorbis decoding? This is a separate license addendum that your company might have chosen not to sign. Look for the ASI codec library files (*.ASI in Windows) in the redistribution directory - are they present?
  6. Is the file in question a true MP3 file? Miles doesn't play MPEG Layer 1 or MPEG Layer 2 files which are often mistakenly labeled MP3 files. Try recompressing the file yourself just to be sure.
