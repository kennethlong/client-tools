This guide will cover how to set up Presets to automatically apply to sounds based on a label query, in this case, all sounds with the label "3D" will be marked as 3d spatialized.

Requisite Terminology

  1. Label
  2. Label Query
  3. Preset
  4. Variable

New Terminology

  1. Persisted Preset: A persisted preset is a preset that is conditionally applied to all sounds, based on a label query. If the sound matches against the label query, then the preset is applied when the sound starts. Much like normal Presets, if the persisted preset is marked as dynamic, it will update every frame due to variable changes.

  2. Perist Preset Event Step: The event step that causes a persisted preset to start taking affect.

Steps

  1. Create a label named "3D"

  2. Create a preset, and set the Spatialization to "3D". In addition, set the Playback Rate to -8.



    Since there isn't an easy way to see that this sound is 3D in Miles Studio, we make the affected sounds pitch down by 8 semitones to make the effect more obvious.

  3. Load a sound, and create the event to play it.

  4. Select the event, and delow the "Start Sound" event step, click "Add New Step". Then, click the up arrow next to the new step's type.



    The arrows next to the step's type in an event allow you to move the event step around within the event itself. In this case, we want this new event step to occur before the sound starts, so we move it up.

  5. Now scroll up, as the new step is now first, and set its type to "Persist Preset".



  6. Set the persist name to "3dpersist", drag the preset you created in step 2 to the Preset field, and set the label query to "3D".



    The name of the persist is used to remove it in the future, should you desire. The remainder of the event step should be clear - we are specifying that the preset created in step 2 be applied to all sounds that have the label "3D", as well as any sounds that start in the future with the label "3D".

  7. Audition the event.
