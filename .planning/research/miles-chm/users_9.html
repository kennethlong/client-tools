Miles Studio is designed to handle multiple sound banks easily, but there are some things you should know.

The most important thing is how your Settings are managed. There can only ever be one set of settings in a game, so you have to designate one bank to manage these. When you do so, this bank must be loaded first. It is completely reasonable to have a bank that *only* contains the settings.

All other banks in your game will reference this "Settings Bank" for UI purposes, and as a result this bank will always be loaded when another bank using it is loaded.

Otherwise, everything is business as usual. Things to know:

  • Cache/Purge sounds can only ever load from a single bank. The bank they reference must have been loaded from code with AIL_add_soundbank at the time the event is executed.

  • Events can reference assets that exist in another bank. Just like above, the bank they reference must have been loaded at the time of use. References such as these do *not* force the other bank to be loaded in Miles Studio at the same time.

  • Renaming assets normally tracks all references and patches them up. Obviously this can not happen with banks that are not loaded. Be sure that you load all sound banks that reference an asset before renaming it.

  • Deletion is treated the same - which is to say, references are not cleared on deletion. To find dangling references within a bank, go to that bank's Properties, and change the combo box to "Dangling References".
