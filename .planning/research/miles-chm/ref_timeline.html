Overview

The timeline is where all event inspection and replay occurs.

  1. List of open captures. The scratch pad is always open.
  2. Clear the scratchpad to empty.
  3. Open a previously saved capture.
  4. Save the current capture to file.
  5. Close the opened capture.
  6. The selected capture's filename.
  7. Capture playback - Rewind, Stop, Play with new random values, Play with recorded random values.
  8. On/Off - Record ("Capture") the capture to a WAV file.
  9. Set the WAV file to record to.
  10. Choose the settings for the replay - this determines what assets to use if they have been specialized via Specialization.
  11. Zoom the timeline view in and out.
  12. Switch between normal and compact view.
  13. The scrub bar for the timeline. The darkness represents the density of events at that location. Looped sections will be tinted green. A glassy pane shows the currently visible section for long timelines.
  14. The timeline view.
  15. The type of details to show regarding the current capture.
  16. Whether to show the captured or replay state in the details and graphs.

Whenever you audition in the tool, the event will always show up on the Scratch Pad. Not everything can be captured, so you won't get System Graphs for the scratch pad.

  1. The track listing of events. Green means only this event will play, purple means the event is muted, red is the event isn't loaded in the tool, dark grey means the event is muted because another event is set to solo play.
  2. The event instances. A hollow square means the event played no sounds. A solid square means the event played a sound, but it was too short to show at the current zoom. Otherwise, the blue line is the length of the sound, the green is the length of the replay, if one exists.

Event Details

  1. Whether to show event details (shown above), or the system graphs (shown below)
  2. The name of the selected event. Click to edit.
  3. The number of sounds started.
  4. A sound that was started. Click to edit.
  5. The select of source data for the current graph. Note that for events played in Miles Studio, the final volume includes any attenuation from the Master Volume in Miles Studio itself.
  6. The duration, as well as the results of the random volume, pitch, and delay. This does not take in to account 3d volume, ramps, or any other source of volume or pitch - only the sources from the Start Sound event step.
  7. The graph for the current selection. The data type is either decibel, semitones, hertz, or distance.
  8. Whether to show the current ramps or variables that affected this sound.
  9. The list of ramps or variables (based on above), by name.
  10. The name of the ramp/variable, whether it affected all sounds or just this sound, and what event caused it.
  11. The graph of the ramp/variable's value over the length of the sound instance.

System Graphs

Graphs show various values over time. These include system information, user ramps, and user variables. These graphs line up with the timeline scrollbar.

  1. The CPU Usage graph. Given as a percent of total CPU time.
  2. The remaining handle memory. This is the memory that the event system uses for managing sounds. Sound data is not included in this.
  3. The number of playing samples.

    More graphs are available than shown.
