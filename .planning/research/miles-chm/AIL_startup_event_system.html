HEVENTSYSTEM AIL_startup_event_system(HDIGDRIVER dig,
S32 command_buf_len,
char* memory_buf,
S32 memory_len);
Description

Initializes the Miles Event system and associates it with an open digital driver.

Parameters
dig [in]

The digital sound driver that this event system should use.
command_buf_len [in]

An optional number of bytes to use for the command buffer. If you pass 0, a reasonable default will be used (currently 5K).
memory_buf [out]

An optional pointer to a memory buffer buffer that the event system will use for all event allocations. Note that the sound data itself is not stored in this buffer - it is only for internal buffers, the command buffer, and instance data. Use 0 to let Miles to allocate this buffer itself.
memory_len [in]

If memory_buf is non-null, then this parameter provides the length. If memory_buf is null, the Miles will allocate this much memory for internal buffers. If both memory_buf and memory_len are null, the Miles will allocate reasonable default (currently 64K).
Return value
return [out]

Returns 0 on startup failure.

Discussion

This function starts up the Miles Event System, which is used to trigger events throughout your game. You call it after AIL_open_digital_driver.
