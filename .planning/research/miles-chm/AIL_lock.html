// Function prototype:

void AIL_lock();
Description

suspends timer service to all MSS drivers and application callback functions until a matching call to AIL_unlock is made.

Return value
return [out]

Discussion

Calls to AIL_lock and AIL_unlock should bracket any sections of application code which must not be interrupted by background MSS timer callbacks. Examples would include non-reentrant application functions which may be called from both the foreground application and an MSS timer, XMIDI, or digital sound callback function.

AIL_lock increments a counter which is decremented by each corresponding call to AIL_unlock. When an AIL_unlock call decrements the counter to zero, MSS timer service is resumed.

Note that this call won't insure that the background thread isn't already running, just that if it has begun yet, it won't run. To guarantee that the background thread hasn't started, use AIL_mutex_lock.
