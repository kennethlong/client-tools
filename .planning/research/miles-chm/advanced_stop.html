Often, you want to start a sound from an event that will loop continuously until some other player or game driven trigger occurs. You can technically just use AIL_end_sample, but that wouldn't get caught by the auditioner, so captured replays wouldn't replay correctly. It is better to use the event system to accomplish this, and we have a few helper functions to get you what you need:

AIL_stop_sound_instances AIL_pause_sound_instances AIL_resume_sound_instances

You can use a standard label query to stop a group of sounds, but you can also target the stop to a specific ID - and this is probably what you'll use the most. 

    // Fire off a laser beam that loops until its turret is destroyed.
    void Laser::StartFiring()
    {
        this->LaserEventId = AIL_enqueue_event_by_name("Effects/Laser");
    }

    // Stop firing when we've been destroyed.
    void Laser::Destroyed()
    {
        AIL_stop_sound_instances(0, this->LaserEventId);
    }
