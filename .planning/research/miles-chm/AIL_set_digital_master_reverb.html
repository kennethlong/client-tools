void AIL_set_digital_master_reverb(HDIGDRIVER dig,
F32 reverb_decay_time,
F32 reverb_predelay,
F32 reverb_damping);
Description

sets the global reverb settings to use.

Parameters
dig [in]

indicates the driver for which the new master reverb settings are to be specified.
reverb_decay_time [in]

specifies the reverb decay time.
reverb_predelay [in]

specifies the delay before the reverb will be heard.
reverb_damping [in]

specifies the high frequency damping that should be performed - this is a low-pass filter performed on the samples that are fed to the reverb engine. The range of this parameter is 0.0 to 2.0, relative to the playback frequency of the sample.
Return value
return [out]

Discussion

The AIL_set_room_type function uses this function to set EAX-compatible room styles.

You need to set a non-zero wet reverb level of any samples that you want to apply global reverb to - use the AIL_set_sample_reverb_levels to do this.

Reverb decay time ordinarily ranges from 0.1 to 20.0 seconds on hardware EAX platforms, with a recommended 0.0 to 3.0-second range in in software-mixed providers. The maximum reverb_predelay value is limited to the value of the DIG_MAX_PREDELAY preference on software-mixed platforms, or 0.1 seconds on hardware platforms. The total reverb duration period is equal to the sum of reverb_predelay and reverb_decay_time.

Global reverb is very inexpensive to apply (about 0.25% - one forth of one percent - per sample).

This function will have no effect with per-voice hardware drivers that don't support reverb in hardware. Currently, this applies only to the MSS_MC_DIRECTSOUND3D value from the MSS_MC_SPEC list, since the EAX providers handle reverb directly and the software-mixing options emulate EAX.
