// Function prototype:

HPROVIDER AIL_digital_output_filter(HDIGDRIVER dig);
Description

returns the output filter (either per-voice or matrix) provider for a specific driver.

Parameters
dig [in]

specifies the HDIGDRIVER to query.
Return value
return [out]

The output filter provider, or 0 if the driver's output format does not require an output filter.

Discussion

Certain output configurations supported by AIL_open_digital_driver, including DirectSound3D, EAX, DTS, Dolby Surround, and SRS Circle Surround(r), are implemented with extension modules called output filters. Output filters can offer additional control options beyond those available from the core MSS API. Most applications will never need to work with output filter controls directly, but those that wish to do so can obtain a handle to the current voice or matrix output filter from AIL_digital_output_filter and access provider- and driver-level properties via the AIL_filter_property and AIL_output_filter_driver_property functions.

Sample-level properties may be accessed through calls to AIL_sample_stage_property with the SP_OUTPUT stage identifier. Sample-level property control is available only for hardware voice filters such as DirectSound3D and EAX, not matrix filters such as Dolby.

You can find a list of configuration-specific properties in Digital Audio Services. See the examms demo program for an example of how to enumerate and query provider-level, driver-level, and sample-level properties.
