Only a few properties are exposed at the moment. In general, the bank properties page is used to find assets that are potentially problematic.

Properties

  • Use Source Control If checked, editing an asset will cause Miles Studio to try and update the file in Perforce.
  • Compile Unreferenced If checked, all non-event assets will be compiled, whether or not they are referenced by any events. This excludes any references due to a Cache/Purge Sounds event step, and if a sound is only referenced via one of those two event steps, the event will be edited on compilation to exclude that reference.

Asset Search

The bank properties has a variety of searches it can do to find potential asset problems within the bank:

  • Unreferenced Assets Displays non-event assets that are not referenced by any event.
  • Dangling References Displays any references an event may have that doesn't exist in the bank.
  • Unlabeled Sounds Displays any events containing Start Sound event steps with no labels assigned to them.
  • Uncached Sounds Shows any sound assets that are never loaded via a cache sounds event step.
  • Sounds Not Ready Lists any sound assets that are not in a completed state, and are thus not ready to play. That is, they still need a source asset, their compression is out of date, or they are still compressing.
