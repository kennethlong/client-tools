F32 X,

Description

sets the three-dimensional position of the 3D listener.

Parameters
dig [in]

indicates the driver representing the 3D listener.
X [in]

specifies the X coordinate (usually imagined as left to right - left is negative, right is positive).
Y [in]

specifies the Y coordinate (usually imagined as down to up - down is negative, up is positive).
Z [in]

specifies the Z coordinate (usually imagined as back to front - back is negative, front is positive).
Return value
return [out]

Discussion

Following DirectSound3D's somewhat-awkward lead, Miles uses the left-handed coordinate system. All of DirectX also uses left-handed coordinates, so passing in coordinates from a DirectX application is simple. OpenGL, on the other hand, cleanly uses right-handed coordinates, so you must invert the sign of the Z parameter before passing it into Miles.
