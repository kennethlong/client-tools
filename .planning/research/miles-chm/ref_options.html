Various options for controlling global settings for Miles Studio are here.

  1. Source control options. Miles Studio doesn't manage passwords - you need to log in with p4 login, P4Win, or P4V prior to using Perforce in Miles Studio.
  2. Commit the Perforce changes. The settings do not take effect until Commit is clicked, or the Studio is restarted.
  3. The location of the various compressors Miles Studio uses.
  4. Browse for the directory for the compressor.
  5. Commit the compressor changes. Settings do not take effect until Commit is clicked, or the Studio is restarted.
  6. Source Directories - see below.
  7. The directory aliases and the location on disc to use for the alias.
  8. Add a new alias and path definition.
  9. The number of speakers Miles Studio uses for output.
  10. The prefix to apply when doing a "Create Event For Sound" action.
  11. If checked, a variety of statistics about a compiled bank will be written to "bankname.msscmp.stats".
  12. Save/Revert the current options to disk. This does not commit the Perforce or Compressor settings!

Source Directories

Source directories are aliases for directories that may move between machines. In short, there's a name and an associated location on disk. If there's a sound that uses that path, instead of using the direct path, it uses the alias. When the bank is loaded on a different machine, the alias can reference a different location, allowing for some flexibility between sound designer's machines.
