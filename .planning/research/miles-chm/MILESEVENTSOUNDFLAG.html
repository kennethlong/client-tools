// Type Definition:
typedef U32 MILESEVENTSOUNDFLAG;
Description

Specifies the status of a sound instance.

Predefined values
MILESEVENT_SOUND_FLAG_MISSING_SOUND 

The event system tried to look up the sound requested from a Start Sound event and couldn't find anything in the loaded banks. [Value: 0x1]
MILESEVENT_SOUND_FLAG_EVICTED 

The sound was evicted due to a sound instance limit being hit. Another sound was selected as being higher priority, and this sound was stopped as a result. This can be the result of either a Label Sound Limit, or a limit on the sound itself. [Value: 0x2]
MILESEVENT_SOUND_FLAG_WAITING_ASYNC 

The sound is pending because the data for it is currently being loaded. The sound will start when sufficient data has been loaded to hopefully avoid a skip. [Value: 0x4]
MILESEVENT_SOUND_FLAG_FAILED_HITCH 

The sound meta data was found, but the sound was not in memory, and the Start Sound event was marked as "Must Be Cached". To prevent this, either clear the flag in the event, which will cause a start delay as the sound data is asynchronously loaded, or specify the sound in a Cache Sounds step prior to attempting to start it. [Value: 0x10]
MILESEVENT_SOUND_FLAG_FAILED_ASYNC 

The sound tried to load and the asynchronous I/O operation failed - most likely either the media was removed during load, or the file was not found. [Value: 0x20]
Discussion

These are the flag values that each sound instance can have. Use AIL_enumerate_sound_instances to retrieve them. Instances may have more than one flag, logically 'or'ed together.
