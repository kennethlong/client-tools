// Function prototype:

S32 MilesStartAsyncThread(U32 spu_index,
void const * spu_info);
Description

starts one of the background Miles SPU workers.

Parameters
spu_index [in]

Specifies the SPU index to use.
spu_info [in]

Specifies the SPU library specific data.
Return value
return [out]

Returns one on success, zero if there was an error.

Discussion

This function starts up a background SPU worker that Miles can use to perform its background tasks.

The spu_index is SPU library specific. For SPU threads, this number is from 0 to 5 and returns to one of 6 SPU threads that Miles can create. For RAW SPUs, this number is from 0 to one less than the number of RAW threads you left over in the sys_spu_initialize function call. For a SPURs task, this number is from 0 to 5 refers to one of the 6 SPUR task ids that Bink can create.

On SPU threads and RAW, the spu_image is a pointer to a sys_spu_image_t structure that you get from loading an SPU image file (an elf) with the sys_spu_image_open function - this elf should be built from the binkspu.a library. The sys_spu_image_t pointer can be free upon return from this function, since the system caches the spu_image data itself.

With SPURs tasks, the spu_info is a pointer to a buffer containing two values. The first value should be a pointer to your global CellSpurs structure. The second value should be a pointer the ELF image itself. Just load the elf into a memory buffer (or use objcopy to build it directly into your PPU application), and then pass a pointer to the start. These pointers must remain valid until you close the Miles async thread (since SPURs will copy the read-only parts of the image down to the SPU whenever it starts running).
