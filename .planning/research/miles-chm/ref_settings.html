Settings

Settings are global definitions, that are designed to centralize names, ramps, blends, limits and more. They will cut down on typos, and they also make Studio use faster (because that you don't have to constantly type the name name over and over).

There are 5 different settings - Ramps, Blends, Variables, Labels, and Limits. Ramps, Blends, and Limits have editable definitions that are loaded in to the event system when the sound bank is loaded - no event needs to be run in order to set them.

  • Defaults are per platform compressions settings so you can easily customize compression without creating child assets.

  • Busses are definitions controlling what samples get mixed together, and what effects to apply to the mixed bus.

  • Ramps are means of interpolating aspects of sounds smoothly over time. Interpolation takes places in the correct units for the value - e.g. linear in loudness for volume, etc. Variable targets are checked on load, and whenever an interpolation is actively happening - meaning if a variable is changed after the interpolation has completed, nothing will happen. For example, you do an Apply Ramp, to target the variable "EffectVolume", and interpolate to it over 1 seconds. If EffectVolume gets changed by the game during the 1 second period, the ramp will pick up those changes. Once the 1 second completes, any further changes will not be recognized.

  • A Blend is effectively a lookup table used to dynamically crossfade and pitch adjust a group of sounds based on a game's input. For each sound, fade in, fade out, and pitch bend ranges are defined. The pitch bend occurs over the entire audible range of the sound - meaning if the input value from the game is between the fade in start value, and the fade out end value, the pitch bend will occur, and be interpolated across that range.

    When using a Blend, add the desired number of sounds in order to a Start Sound event step, and set the select type to Blended and select the desired Blend.

  • Limits are upper bounds on the number of sounds that can be playing, per label. If a sound contains the label, then it applies to the limit for that label - meaning a sound can effectively count "twice". Only one limit can be effective at any given time, and when the sound bank is loaded, the first limit in the list is applied by default.

  • Variables are values that are set by the game. They are defined here primarily for reference, and can be used in various places in Miles Studio.

  • Labels are categories to assign sounds to. Sounds can have more than one label on them - another common term for them are tags. For coarse management of labels, you can drag an event to a label and it will be assigned to each Start Sound in the event. For more precise management, the start sound event can be used. Labels are used in one of two different ways. One is for sound limiting purposes, as outlined above. The other is for querying the list of playing sounds via Label Queries.
