Description

describes the stage in the sample pipeline to install the filter on.

Predefined values
SP_FILTER_0 

Specifies the first of eight filter provider stages. Once set, the filter in each slot will be executed before each buffer from the HSAMPLE is copied into the mixing buffer. You will almost always use the SP_FILTER_x stages to install and communicate with the Miles sample pipeline filters.
SP_FILTER_1 

Filter pipeline stage 1.
SP_FILTER_2 

Filter pipeline stage 2.
SP_FILTER_3 

Filter pipeline stage 3.
SP_FILTER_4 

Filter pipeline stage 4.
SP_FILTER_5 

Filter pipeline stage 5.
SP_FILTER_6 

Filter pipeline stage 6.
SP_FILTER_7 

Filter pipeline stage 7.
SP_OUTPUT 

Intended to be used to address sample-level properties on output filters.
SP_MERGE 

Specifies the merge provider stage. The merge provider is called to mix each HSAMPLE's current address into the internal MSS mixing build buffer.
