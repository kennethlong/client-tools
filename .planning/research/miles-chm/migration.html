Migrating From Miles 8

In Miles 9, the format of Sound Banks was fundamentally changed. These changes allowed us to add many new features and optimiziations - loading is now simpler, multiple users can edit a Sound Bank at once, deployment is simpler, etc. Moving to Miles 9 will be a little work, but it will totally be worth it!

For Miles 9, sound banks are compiled in to a rapid-load format. This is the only format the runtime can load, and it does so in a handful of allocations. No more confusion between editing banks and deployed "child" banks.

What does this mean for you? Well, the bad news is that a few of the things that were available in Miles 8 sound banks are no longer available. We're pretty sure no one used them, and hopefully the benefits dramatically outweight the drawbacks. Without further ado:

  • Compression/platform overrides are done via a new system. If you want to specialize the deployment of an asset to a target platform, you need to use Asset Reference. This is far easier to manage, but a little less flexible.

  • AIL_find_event isn't really needed anymore. The common usage pattern was to use this prior to a AIL_enqueue_event, but really you just want to enqueue by name. This is needed for the auditioner anyway, so if at all possible, use AIL_enqueue_event_by_name. If you're worried about speed, the assets are sorted when compiled, and searched via binary search.

  • Names include the soundbank. Previously there were places you could get away with just specifying the name of an event inside a bank, e.g. "MyList/EventName", but now the soundbank name is always required: "MyBank/MyFolder/EventName".

  • Sound flag enum changes. Some errors were not specific enough when enumerating sounds, these have been clarified.

  • AIL_add_soundbank requires a name. This is for auditioner purposes - it should be what your soundbank is named, prior to filename changes due to compilation. See docs for AIL_add_soundbank.

  • Externally deployed sounds are not handled the same way at all. It was difficult in Miles 8 to assure that the sounds would be found, and with the new async loading system, the runtime needed metadata that just isn't available if all you have is a filename. The new system requires that you supply Miles with the information necessary to load the file correctly. See the eventexternal example.

  • The error handler now has a bit more information with it. These are also captured by the auditioner.

  • The giant 2d channel<->speaker matrix is gone. If you need that level of control, your programmer can do it.

It's not all bad news, however. A few things are markedly easier in Miles 9:

New Stuff To Use

  • Crossfading is super easy now! Control Sounds allows you to fade a sound out linearly in loudness, and Start Sounds allows you to fade a sound in linearly in loudness.

  • High Level variables. For most properties you can enter a variable instead of a number, and the runtime will poll this value for the relevant number.

  • Sounds can be controlled via time instead of just byte offsets. This is limited to sounds that this works well for - Bink Audio and WAVs, for instead. Most compressed format's don't allow for such exact seeking.

  • Random No Repeat, Random List in Start Sound. Everyone asks for it, and now you have it.

  • Blended Sounds. These are primarily designed for car or other engine sounds, and allow you to play multiple sounds simultaneously, crossfading and pitch bending between them based on an input variable set by the game.

  • Ramps. Smoothly adjust the volume, pitch, wet level, or low pass cutoff for groups of sounds.

  • Folders. Actual folders, not weird list things that save all at once for no apparent reason.

  • Undo! Ctrl-Z mistakes for an easier, less aggravated editing experience.

  • Alter filters in a preset without re-applying existing filters. You can now ignore a filter slot in a preset, allowing presets to independently alter filter slots.

  • Per sample 3d falloff. Some sounds you want to fall off more quickly than others, and now you can do that.

  • And last but not least, the Miles Sound Studio asuditioner, which lets you capture game audio traces, modify them, and then hot-load the data back into the game!
