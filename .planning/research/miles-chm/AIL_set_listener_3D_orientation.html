Description

specifies the orientation of the 3D listener.

Parameters
dig [in]

indicates the driver representing the 3D listener.
X_face [in]

indicates the X coordinate of the facing vector.
Y_face [in]

indicates the Y coordinate of the facing vector.
Z_face [in]

indicates the Z coordinate of the facing vector.
X_up [in]

indicates the X coordinate of the upward vector.
Y_up [in]

indicates the Y coordinate of the upward vector.
Z_up [in]

indicates the Z coordinate of the upward vector.
Return value
return [out]

Discussion

Orientation is specified with two vectors. The facing vector tells which direction the object is facing (where the listener's 'nose' is pointing), and the upward vector is the vector pointing upward out of the top of the 3D object (out the top of the listener's head, for example).

The facing vector and upward vector should always be at right angles to each other. They will be normalized internally by MSS; you do not have to normalize them yourself.

Following DirectSound3D's somewhat-awkward lead, Miles uses the left-handed coordinate system. All of DirectX also uses left-handed coordinates, so passing in coordinates from a DirectX application is simple. OpenGL, on the other hand, cleanly uses right-handed coordinates, so you must invert the sign of the Z parameter before passing it into Miles.
