// Function prototype:

typedef F32 AILCALLBACK AILFALLOFFCB(HSAMPLE sample,
F32 distance,
F32 rolloff_factor,
F32 min_distance,
F32 max_distance);
Description

specifies a user-defined 3D distance-based falloff function callback.

Parameters
sample [in]

indicates the HSAMPLE for which the callback is being issued.
distance [in]

indicates the distance (in user units) between the emitter and listener.
rolloff_factor [in]

indicates the rolloff factor. This value defaults to 1.0, but can be altered by calling AIL_set_3D_rolloff_factor.
min_distance [in]

indicates the sample's minimum distance, as defined by the most recent call to AIL_set_sample_3D_distances.
max_distance [in]

indicates the sample's maximum distance, as defined by the most recent call to AIL_set_sample_3D_distances.
Return value
return [out]

This callback should return a value from 0.0 to 1.0, which will be treated as a linear amplitude scalar during the system's internal volume calculations. The equivalent gain in dB will be 20 * log10 (value) -- for example, a 6-dB power attenuation value would correspond to a returned value of 0.5.

Discussion

Note that falloff-function callbacks are invoked during background HSAMPLE processing. You should be careful not to make any API calls from within a falloff-function callback that could affect the state of the HSAMPLE or its corresponding HDIGDRIVER. If you do anything other than basic 3D state queries, undefined behavior may result. If in doubt about whether a particular action may safely be performed from within a falloff-function callback, contact RAD Game Tools for assistance.
