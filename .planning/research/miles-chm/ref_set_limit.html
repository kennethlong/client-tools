Contains a list of "Limit Entries", each of which set the maximum number of sounds that can play for a label.

  1. A label to limit.
  2. Remove the limit for this label.
  3. Add a new label for limiting.

When a sound starts, it contains a list of labels that are assigned to it (via Start Sound. For each label, the corresponding Limit Entry is found, and if there are too many sounds playing with that label, one will be selected to not play. The removed sound can be either the new sound, or an existing sound. As an example:

  • We have an event that starts a sound "Gunshot", with the label "Effect". We leave all other settings in the Start Sound event as default.

    We've also created a limit with a limit entry defining "Effect" to only allow 2 concurrently playing sounds.

    We then queue that event 3 times in a row, rapidly enough that the first sound has not had a chance to end.

    When the 3rd event tries to play, the limit entry is checked, and it is determined that a sound has to stop. This will either be one of the 2 existing sounds, or the new sound that is trying to play.

    By default, the start sound event step specifies "Age" for the parameter "If Limited, evict:". (see Start Sound. As a result, the oldest sound is chosen to stop, and the new sound is allowed to begin.

This occurs for each label listed on a sound, so even though the first label listed might not bump up against a limit, the next one might.

If a limit entry does not exist for a label, then that label is not limited.

If a start sound event step has no labels listed, then the sound is not limited. To find unlabeled sounds, go to the bank properties and choose "Uncategorized Sounds".
