<objective>
Stand up the throwaway, uncommitted scratch `Debug|x64` compile-only harness (D-01) — the
compiler-driven worklist generator that every later fix-wave samples against. It compiles the
in-scope engine-lib translation units to x64 with `cl /c` (no link → no x64 third-party libs
needed) and lets `cl.exe` emit the real C4235 (inline asm) + C4311/C4312/C4244 (truncation, as
errors) worklist. This converts the audit from grep+eyeballs into a deterministic compiler worklist.

Purpose: This is the Wave-1 enabler (the Wave-0 deliverable for the fix waves). BITS-02 truncation
enumeration (plan 04) and the BITS-01 asm-survivor proof (plans 02–03) are validated against this harness;
the phase gate (plan 06) runs the full sweep through it. The harness MUST NOT touch the committed
`swg.sln`/`.vcxproj` (the committed x64 platform-add is Phase 33 scope per D-01).

The cross-AI review (#4) flagged the harness as the integrity backbone of the whole phase: a
"first-cut" manifest can green-gate large latent defect populations in unlisted TUs; relative include
extraction can silently fail (TU skips → false-clean); and EXPLICIT casts suppress C4311, so the harness
is authoritative ONLY for implicit truncations. The checker additionally flagged that every downstream
plan's `<verify>` invokes `-SingleTu`/`-Scope`/`-CastAudit` modes the driver MUST implement. This plan
now hardens against all of it: an exhaustive manifest derived from the ClCompile graph, a positive "one
clean TU compiles" proof, an explicit-cast grep audit, and a fully-specified parameter interface.

Output: An uncommitted, gitignored `src/build/win32/x64-scratch/` directory containing the props,
the PowerShell driver (with all three verify-modes), and the exhaustive in-scope TU manifest — plus a
`.gitignore` line so it never enters a commit.
</objective>
