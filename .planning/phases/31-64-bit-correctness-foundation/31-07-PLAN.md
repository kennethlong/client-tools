<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| node mark pointer round-trip → pathfinding memory safety | PathSearch stores a live `PathSearchNode*` in a mark field and dereferences it later; on x64 a 32-bit int field truncates the pointer → use-after-free / corruption. This is the highest-severity item (memory safety, not diagnostics). |
| window userdata pointer → UI object dispatch | StatusWindow stores `this` in the window-userdata slot and reads it back in the window proc; truncating it on x64 dispatches messages to a corrupt pointer. |
| SSE skinning lane port → visual/skinning correctness | A non-equivalent _mm_* translation silently corrupts skinned geometry (no compile error); the _DEBUG oracle is the trip-wire. |
| shared template header (VoidBindSecond.h) → ABI of all consumers | Touching a shared header changes codegen for every consumer; the full 5-target build (plan 31-06 Task 2) must rebuild all plugin DLLs (AGENTS.md ABI-cascade trap). |
| time_t width at a serialization boundary → 32-bit/x64 wire parity | Widening a time_t that crosses the wire would break parity with the shipped 32-bit server; the audit pins serialized sites to a fixed width. |
