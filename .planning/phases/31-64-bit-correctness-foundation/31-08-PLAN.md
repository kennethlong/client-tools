## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AutoDelta* serialized count width → 32-bit/x64 wire parity | The object-delta network path serializes container/command counts; widening a count to 8 bytes on x64 would desync the wire from the shipped 32-bit server. Pinning to uint32_t keeps the bytes identical — the count was always a 4-byte field. |
| two divergent header copies → consumer correctness | src/shared and include/Archive copies are both consumed via different include paths; fixing only one silently leaves half the consumers with the x64-broken header. |
| count overflow on narrowing put | A container with > UINT32_MAX entries is not representable in the existing wire format; an unguarded narrowing cast would silently truncate. The assert is the trip-wire. |
| template member-type change → consumer recompile | Retyping baselineCommandCount in a template header recompiles all consumers; the full 5-target build (31-06 Task 2) rebuilds them — no stale-object ABI risk. |
