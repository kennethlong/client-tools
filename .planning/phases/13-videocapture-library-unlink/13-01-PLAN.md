<objective>
Land the ATOMIC link-unit removal of the VideoCapture middleware so SwgClient no longer
references or links any capture-family symbol. This is the linchpin plan (RESEARCH edit-order
steps 1-3) and the DECRUFT-04 build gate: it must leave the SwgClient Debug AND Release link
with zero unresolved externals under the `/FORCE`-armed linker.

Three things land together IN ONE ATOMIC TASK (Task 2) so the tree never has a reference without
a provider, even mid-step (RESEARCH "atomic-delete linchpin", D-01):
  1. Remove the ONE live caller — `CuiIoWin.cpp` `#include` + the `#if PRODUCTION==0` run() block.
  2. Drop the wrapper from compilation — `clientGraphics.vcxproj` ClCompile/ClInclude + videocapture
     include path (3 configs) — then delete the wrapper files.
  3. Drop the lib link inputs — 15 capture lib tokens + 8 videocapture library-search paths from
     all 3 `SwgClient.vcxproj` `<Link>` configs (Debug/Optimized/Release).

These three steps are ONE task / ONE commit-unit (not three tasks) precisely so an autonomous
executor CANNOT commit a half-state (callers gone but lib tokens present → /FORCE-masked LNK2019;
or tokens gone but a search path dangling → LNK1181).

Purpose: Satisfy DECRUFT-04 criterion #2 (clean Debug+Release link) and remove the only live
recorder code path. The dead `#if 0` source residue, the `.rsp`/editor `.vcxproj` purge, and the
vendored-tree delete are handled by Plans 02 and 03 — they cannot affect this link.
Output: A SwgClient that links clean with the capture family gone (link-log grep == 0).
</objective>
