## writeVarByName mechanics (per 17-04 SUMMARY + 17-03 SUMMARY)

`writeVarByName` is a local lambda at Direct3d11_StaticShaderData.cpp line ~794 that:
  - looks up `varName` in the cached reflected cbuffer layout (m_reflectedCbufferLayouts via Plan 17-02's cache)
  - if found, writes the XMFLOAT4 value at `var.StartOffset` into the staging byte buffer sized to `layout.TotalSize`
  - returns true on hit, false on miss
  - bounds-checked against var.Size

For nested struct paths like `userConstants.lighting.material.diffuse`, D3D11 reflection uses dot syntax in `GetVariableByName`. The lookup string passed to writeVarByName should match the canonical reflection-side dotted path. The executor confirms the exact dotted form from Path A source-read or Path B discovery-dump output.
