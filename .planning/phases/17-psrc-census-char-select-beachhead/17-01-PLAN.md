Classification rule — UPDATED IN ROUND 3 to match ShaderBuilder's first-line semantics
(ShaderBuilder PixelShaderProgramView.cpp:301-308 classifies the FIRST LINE, not byte 0):
  R3-01c CONTRACT — the classifier MUST do this:
    1. Skip leading whitespace bytes (\r \n \t and space — ' ', 0x20).
    2. Skip a single optional UTF-8 BOM (the 3-byte sequence 0xEF 0xBB 0xBF) at the very start;
       if the BOM is followed by whitespace, also skip that whitespace.
    3. From the resulting offset, copy up to 7 bytes into a scratch buffer; lowercase the ASCII
       letters; strict-compare to "//hlsl " (with the trailing space).
    4. If the 7-byte compare succeeds -> HLSL; the profile token is scanned from `offset + 7`
       forward, skipping whitespace, taking the next non-whitespace token (cap 31 chars).
    5. Else -> asm.
  PSRC sources observed in the v2.2 research synthesis are typically pure ASCII with no BOM,
  but the BOM/newline-prefixed case is not impossible and must not silently misclassify.

PSRC length sanity cap (HIGH-5/LOW-3, ASVS V5 / T-17-01):
  Conservative cap = 1 MiB (1024 * 1024 bytes). If iff.getChunkLengthLeft(sizeof(char)) exceeds the cap,
  log a WARNING via DEBUG_REPORT_LOG_PRINT, do NOT allocate (leave m_psrcText=nullptr, m_psrcLen=0),
  exitChunk(true) to skip the remaining bytes, and continue to TAG_PEXE. Bounds the worst-case
  unbounded alloc from a corrupt asset. Char-select PSRCs are far below this cap (max observed
  in the v2.2 research synthesis is ~30 KB).
