<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → server | `left_cfg`/`right_cfg`/`path` in the request body are request-controlled filesystem paths the server opens |
| network → uvicorn bind | the listening socket's bind address determines remote reachability |
| installs.toml → server | a tool-local config supplies cfg paths to the picker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-03-01 | Tampering / Info disclosure | path traversal / arbitrary file read via `left_cfg`/`right_cfg`/`path` | mitigate | Validate cfg paths exist+readable before open → `HTTPException(400)` on failure; the `path` drill-in resolves only against `safe_virtual_key`-canonical keys already in the merged tree (interior `..`/drive/UNC rejected by Phase-28 hardening) — a non-tree path is a not-found, not an open. Residual arbitrary-cfg-open is bounded by the localhost-only bind (T-29-03-03) and accepted for a single-user local diagnostic tool. |
| T-29-03-02 | DoS | malformed/oversized archive or huge entry count amplified through a request | mitigate | Per-side build reuses the Phase-28 bounds preflight + `MAX_ENTRIES` + per-node fault isolation (via the cached merge); a bad archive degrades a row and is surfaced in `summary.node_errors` — it never 500s or aborts the response. |
| T-29-03-03 | Elevation / remote exposure | uvicorn bound to 0.0.0.0 would expose the file-reading tool to the network | mitigate | `__main__.py` binds `127.0.0.1` ONLY; test asserts no `0.0.0.0` literal. Single-user localhost posture. |
| T-29-03-04 | Info disclosure | a user-local `installs.toml` with absolute install paths committed to git | mitigate | Track only `installs.toml.example`; the real `installs.toml` is user-local (add to `.gitignore` if created). The loader treats an absent config as an empty list. |
</threat_model>
