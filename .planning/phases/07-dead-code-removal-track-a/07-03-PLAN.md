<objective>
Complete Phase 7 by removing Vivox voice chat (CLEAN-03) and XPCOM/Mozilla browser (CLEAN-04) at the source level, then dropping the `/Zc:wchar_t-` compiler flag (the critical gate for Phase 9's STL swap, per ROADMAP.md and STATE.md). This plan is the largest of Phase 7 because both removals carry hidden cascade edits across 18 files identified in PATTERNS.md.

The plan is split into three numbered sub-steps with one final boot verify:

- **3a — Vivox voice chat** (Tasks 1-3): Delete Vivox source files, edit 11 cascade files, unlink CMake graph, remove voice preference keys from CuiPreferences. Intermediate build verify.
- **3b — XPCOM browser** (Tasks 4-6): Delete XPCOM source files, edit 7 cascade files, remove the inline `libMozilla` target from `src/engine/client/library/CMakeLists.txt`, remove all Mozilla DLL POST_BUILD copies. Intermediate build verify.
- **3c — `/Zc:wchar_t-` removal** (Task 7): Remove the flag from BOTH line 43 (CMAKE_CXX_FLAGS_DEBUG) AND line 44 (CMAKE_CXX_FLAGS_RELEASE) of `src/CMakeLists.txt`. Preserve STLPort's own `/Zc:wchar_t-` per STLPort 4.5.3 ABI. Intermediate build verify.
- **Final boot verify** (Task 8): Manual character-select boot against VM, plus dumpbin import-table check.

Purpose: This is the high-cascade work. Voice chat in v1 was disabled with ~20 `ms_voiceChatManagerInstalled` guards (per STATE.md); the browser was stubbed (`libMozilla::init` returning true). Both still pulled their headers and DLLs into the build. After this plan, neither feature has any source, library, or DLL footprint, and the wchar_t ABI boundary that XPCOM's `PRUnichar` required is gone. Phase 9 STL-03 (`Unicode::unicode_char_t = wchar_t`) becomes mechanical after this gate.

This plan is `autonomous: false` because the final boot verify is a `checkpoint:human-verify` and because the intermediate build verifies between sub-steps may surface unforeseen cascade misses requiring a checkpoint to triage rather than blind-edit additional files.
