<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> CMake graph | Deleting a directory referenced by `include_directories()` produces a configure-time error if the include path is still emitted to the compiler |
| filesystem -> source `#include` | A `.cpp` that still `#include`s a deleted header fails to compile |
| build artifact -> live game | A clean build that nonetheless fails to boot indicates a runtime regression beyond the build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | clientGame `include_directories()` | mitigate | Remove the trackIR include path on the same step as the directory delete; verify with `Select-String 'trackIR' src/engine/client/library/clientGame/src/CMakeLists.txt` returning empty |
| T-07-01-02 | Tampering | `ClientHeadTracking.cpp` (still in clientGame) | accept | PATTERNS.md confirms `ClientHeadTracking.cpp` `#include "NPClient.h"` will become a missing-header compile error after delete; if build fails, that file must be excluded from the source list as part of this plan (cascade noted in acceptance) |
| T-07-01-03 | Denial of Service | Live client boot | mitigate | Mandatory boot verify against the SWGSource VM at 192.168.1.200 — must reach character select, not just compile |
| T-07-01-04 | Tampering (build artifact) | `SwgClientSetup/` orphan launcher | accept | No CMake target references it; deletion has zero build impact (PATTERNS.md verified) |
| T-07-01-05 | Information Disclosure | Git history | accept | Files are tracked in git; deletion is recoverable from history if a future need arises |
</threat_model>
