<objective>
Sever the CMake graph's connection to three already-disabled middleware features:
- **lcdui_src** — Logitech G15 LCD support (`disableG15Lcd=true` in client.cfg, dead at runtime)
- **Bink** — cinematics video codec (cutscene flags off; `binkw32.dll` not loaded)
- **VideoCapture** — `PRODUCTION==0` debug capture command, replaced by `SwgVideoCapture_stub.cpp` in v1

This plan only edits CMakeLists.txt and deletes Find*.cmake modules. **No source `.cpp`/`.h` files are deleted** — the Bink subdirectory under `src/engine/client/library/clientGraphics/src/Bink/` is left in place but no longer compiled (it is removed from `SHARED_SOURCES` and the include path). Source removal of Bink and the lcdui prebuilt is out of scope for v2 entirely; CMake graph removal is sufficient for CLEAN-02.

Purpose: Validate the multi-CMakeLists.txt unlink pattern (Pattern A from PATTERNS.md) on three relatively low-cascade targets before applying it to the high-cascade Vivox and XPCOM removals in 07-03. Each Find*.cmake module is a single point of truth for its three downstream consumers (root CMakeLists.txt, the using library's CMakeLists.txt, and SwgClient's POST_BUILD copy step).

Output:
- Three Find*.cmake modules deleted
- Three `find_package(... REQUIRED)` calls removed from `src/CMakeLists.txt` (lines 54, 59, 60)
- Six downstream CMakeLists.txt files cleaned of the corresponding `${X_INCLUDE_DIR}` / `${X_LIBRARY}` / `${X_LIBRARIES}` references
- `binkw32.dll` POST_BUILD copy removed from SwgClient's CMakeLists.txt
- Bink source files removed from clientGraphics SHARED_SOURCES list
- Clean Debug build of `SwgClient_d.exe`
- Boot verify pass against the SWGSource VM
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `find_package` -> consumer CMakeLists | Removing `find_package(X REQUIRED)` while a downstream `${X_LIBRARY}` reference remains causes the linker to receive an empty token, which on MSVC manifests as a confusing link error |
| Source list -> compile graph | Bink source files left in `SHARED_SOURCES` after `${BINK_INCLUDE_DIR}` is removed will fail to compile (cannot find `bink.h`) |
| POST_BUILD step -> staging dir | Stale `copy_if_different` lines for deleted DLLs cause configure-time errors if the source DLL is also removed; here we keep the source DLL but remove the copy step (DLL is unused at runtime anyway) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | clientGraphics SHARED_SOURCES | mitigate | When removing `find_package(Bink)`, also remove the 6 Bink source-file entries on lines 3-8 of clientGraphics CMakeLists.txt; verified by `Select-String -Pattern 'Bink/'` returning zero hits in that file |
| T-07-02-02 | Tampering | SwgClient POST_BUILD chain | mitigate | Remove `binkw32.dll` copy line (line 208) — staging dir no longer carries unused DLL; SwgClient's `${VIDEOCAPTURE_LIBRARY}` link entry (line 162) and `${LOGITECHLCD_LIBRARY}` (line 164) must also be removed in the same pass |
| T-07-02-03 | Denial of Service | Live boot | mitigate | Mandatory boot verify against VM — features were already disabled at config level, so removing them from the link graph should be a no-op; if boot fails this is a build-system regression |
| T-07-02-04 | Information Disclosure | Find*.cmake modules in source control | accept | Deleted modules remain in git history, recoverable if Bink/lcdui/VideoCapture are ever revived |
| T-07-02-05 | Tampering (variable expansion) | clientGame line 818 `${VIDEOCAPTURE_ROOT}` | mitigate | After removing `find_package(VideoCapture)`, `${VIDEOCAPTURE_ROOT}` becomes undefined; remove the reference from clientGame's `include_directories()` (PATTERNS.md Step 2-C explicit) |
</threat_model>
