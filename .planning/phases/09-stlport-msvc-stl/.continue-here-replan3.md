### Decoding the FATAL

`format arg out of range: value/max 1793887061/16` is from the engine's
`FormatTokenizer` / `Smart_String_Format` printf-format checker. It thinks it
is processing a `%N$` positional format token where `N` is the argument
position — `1793887061` is `0x6AEDD955` interpreted as a position index.
The "max 16" is the formatter's positional-arg ceiling. The garbage value
points at **stack/va_list misalignment** between the EXE-side caller and
whatever supplied the format string + args.

### Decoding the callstack

| Caller frame | Address | Module guess |
|---|---|---|
| FATAL fire site | `0x01A8DBF1` | EXE — engine `FormatTokenizer` |
| caller 1, 2, 5, 6 | `0x6BEAB2D3`, `0x6BE9678B`, `0x6BE99C87`, `0x6BE992E9` | **DLL-X loaded at `~0x6BE90000` — NOT one of gl05/06/07 (image base `0x62A00000`), NOT dpvs/dpvsd (`0x10000000`), NOT mss32 (`0x21100000`), NOT binkw32/qt-mt334/nspr4 (their image bases). Likely an ASLR-relocated runtime-loaded module — candidates include a system DLL (`d3d9.dll`, `dwmapi.dll`, `uxtheme.dll`) loaded as part of graphics-plugin static-link, or a SWGSource-shipped runtime DLL not tracked in our manifest.** |
| caller 3, 4, 7, 8 | `0x01A2EA35`, `0x01A77A80`, `0x01A2C619`, `0x01A36620` | EXE — engine WARNING/FATAL recursion |
| caller 9, 10 | `0x00DE4A38`, `0x00DE2ACA` | EXE — main entry chain |
| caller 11–14 | `0x0208B52D`, `0x0208B43A`, `0x0208B2DD`, `0x0208B598` | EXE — `WinMain` / engine boot |
| caller 15–17 | `0x76E15D49`, `0x77CBD83B`, `0x77CBD7C1` | Windows system DLLs (RtlExitUserThread chain) |

DLL-X is calling into the EXE's WARNING/FATAL emit API, the EXE's logger
formats a string with a vararg list supplied by DLL-X, and the format
checker reads a garbage arg position. This is the signature of a **runtime
ABI break at the format-string / vararg / calling-convention layer between
the v145 EXE and the 2016 SWGSource v3.0 DLLs.**
