# Risk model: these libs are statically linked into client + tools. If any
# PUBLIC header exposes std::string / std::vector / std::map / std::set /
# std::list / std::deque in a function signature or class/struct member,
# the container layout becomes part of the link-time ABI even though the
# final link is static -- STLPort layout differs from MSVC for all of these
# (std::string COW vs SSO; std::vector pointer-triple-vs-other; node alloc
# patterns). Phase 9 changes the layout; if a public header commits to a
# layout, every consumer must be rebuilt against the new layout -- which is
# exactly what Phase 9 does, but only if the surface is documented + audited.
#
