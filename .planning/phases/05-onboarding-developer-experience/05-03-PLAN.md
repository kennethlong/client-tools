## P2.03 — swg-main LoginServer auth bypass mechanism

### Background
1-2 sentences: the old SOE Station auth path (validateStationKey=1 → session server)
no longer exists. swg-main has replaced it.

### swg-main auth decision tree

Describe the logic in validateClient() as a numbered decision path:
1. Client sends LoginClientId message with username + password/key.
2. Server receives LoginClientId → calls validateClient(id, key).
3. validateClient reads ConfigLoginServer::getExternalAuthUrl().
4. If externalAuthUrl is EMPTY (default): authOK = 1 — any credentials accepted.
5. If externalAuthUrl is set and useJsonWebApi=true: POST JSON to auth endpoint.
6. If externalAuthUrl is set and useJsonWebApi=false: POST form-encoded to auth endpoint.
7. If authOK: calls LoginServer::onValidateClient() → triggers enum cluster reply.

### ConfigLoginServer defaults (swg-main)

Table of relevant config keys:
| Key | Default | Notes |
|-----|---------|-------|
| validateStationKey | false | Dead key — old Station validation; code is commented out |
| doSessionLogin | false | Dead key |
| externalAuthURL | "" (empty) | When empty: all logins succeed unconditionally |
| useJsonWebApi | false | Only relevant when externalAuthURL is set |
| clientServicePort | 44453 | Client connects here |

### Historical whitengold config (for reference only)

Note the values in exe/linux/loginServer.cfg:
  validateStationKey=1  (live SOE setting — bypassed in swg-main by design)
  sessionServer=sdt-session1.station.sony.com:3000  (defunct; endpoint no longer exists)
  DSN=sdswgp5b  (internal Oracle DSN — historical reference only)

Explicitly state: these values are historical artifacts. They are not live credentials.

### Client-side requirements for Phase 6

  loginServerAddress0=<swg-main VM IP or 127.0.0.1>
  loginServerPort0=44453

  # gameFeatures=15 (all SKU bits; set elsewhere in client.cfg)

Authentication: any username/password pair will be accepted by swg-main when
externalAuthURL is empty (the default for StellaBellum-style VMs).

### Phase 6 implications

- The auth bypass is already in place in swg-main; no server changes needed.
- Client must NOT send a [Station] sessionId — that triggers the old Station SSO

- Phase 6 should verify that loginServerAddress0 in client.cfg resolves to the VM.
