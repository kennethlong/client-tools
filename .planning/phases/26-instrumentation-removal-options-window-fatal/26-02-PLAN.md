<objective>
Resolve HARD-04: opening the Options window must not FATAL. The crash came from
`checkShowToolbarCommandCooldownTimer` — a CodeData/.ui-widget lookup added by feature commit
`d1b3c0eaf` whose widget is absent from the shipped (SWGSource v3.0-era) Options `.ui` asset.
A required `getCodeDataObject` lookup of an absent widget hits `FATAL(!optional, ...)` in
CuiMediator and crashes the whole Options window.

IMPORTANT — verify reality first: a prior commit (`5fce7bb83`, "fix Options-window crash")
already made this lookup optional (`getCodeDataObject(..., true)` + `if (checkbox)` guard at
SwgCuiOptUi.cpp:224-226), with the rationale documented inline at :219-222. This plan's primary
job is therefore to VERIFY the fix actually holds at runtime under both renderers and to AUDIT
that no OTHER required lookup in the same page FATALs on the shipped widget set — not to
re-implement a fix that already exists. Only apply a code change if the audit or the runtime
smoke surfaces a still-FATALing widget.

Purpose: Confirm HARD-04 is closed against runtime reality (functional truth, not artifact
assumption), and harden any remaining required lookup that the shipped .ui cannot satisfy.
Output: Options window opens without FATAL under rasterMajor=5 and rasterMajor=11.
</objective>

  - FATAL (!optional, ("Unable to find CodeData object '%s' ...")) at CuiMediator.h:280.

:353 (checkAutoSortInventoryContents), :361 (checkAutoSortDataPadContents).
