<objective>
Land the residual D3DX removal on the D3D9 **asm** compile path — the last D3DX compile dependency (the HLSL path already migrated to D3DCompile in Phase 32-02). Port `D3DXAssembleShader` (`Direct3d9_VertexShaderData.cpp`, currently :760) to `D3DAssemble` (d3dcompiler_47) via the static-local-cached `GetProcAddress` shape validated in Phase 32-01, then resolve the Fix-A SEH guard disposition. This is the **D3DX-removal precondition** (D-04/D-05) — `d3dx9` has no x64 static lib, so the asm path MUST be off D3DX before the x64 link in Plan 05 can be clean. The work is done on the still-32-bit tree and re-validated against the 32-bit gl05 reference, so nothing regresses while x64 is not yet building.

Background: The Phase 32-01 D3DAssemble dialect gate already PASSED byte-identical (6/6 shaders incl. `lit` + `if b#`, `CreateVertexShader` S_OK — STATE.md), so this is a mechanical `GetProcAddress`-on-undecorated-export swap, not an unproven port. gl11's `[P19VSFALLBACK]` substitution does NOT transfer — D3D9 gl05/06/07 natively execute the assembled vs_1_1/vs_2_0 bytecode, so the asm must be *genuinely assembled* (D-05 finding).

Purpose: Clear the final D3DX compile symbol so the x64 link is achievable; keep both renderers rendering on the 32-bit gate.
Output: the ported asm call site (off D3DX); render-clean //asm shaders under rasterMajor=5; the Fix-A disposition recorded.
