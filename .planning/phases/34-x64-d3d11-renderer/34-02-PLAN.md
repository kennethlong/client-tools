<!-- Runtime facts the executor needs. No exploration required. -->

Renderer select: `rasterMajor` in the active .cfg picks the plugin DLL (NOT the DX version):
5=gl05 (D3D9), 11=gl11 (D3D11). `rasterMajor=9` is FATAL (Graphics.cpp:223 — DX9 is value 5).

Which exe reads which cfg: SwgClient_d.exe (Debug) → client_d.cfg. The x64 client lives in
stage-x64/ (distinct from the shipped 32-bit stage/). So for the x64 gl11 boot:
boot stage-x64\SwgClient_d.exe with stage-x64\client_d.cfg set to rasterMajor=11, and confirm
exe + cfg + rasterMajor + gl11_d.dll all line up IN stage-x64/ before declaring a verify.

cfg-safety: NEVER write a .cfg with PowerShell Set-Content/Out-File — PS 5.1 adds a UTF-8 BOM that
crashes the Release client at boot (Debug masks it). Use the Edit tool or bash (both BOM-safe).
Diagnose with `head -c 8 stage-x64/client_d.cfg | xxd` — clean starts `23 20` (`# `), BOM starts
`ef bb bf`.

RenderDoc CLI (CAN capture D3D11 — both sides of this A/B are D3D11):
`D:\Code\renderdoc-mcp\v0.3.0\renderdoc-mcp-windows-x64-v0.3.0\bin\renderdoc-cli.exe`
Subcommands: `capture EXE -o PATH`, `draws [--filter]`, `pipeline -e EID`, `shader vs|ps -e EID`,
`pixel X Y -e EID`, `debug pixel X Y -e EID --trace`, `export-rt`.

Canonical regression triad (D-02): dressed character-select + Tatooine + cantina interior.
Representative draws to diff: a dot3/bump character surface (dressed NPC at char-select), a
terrain-blend draw (Tatooine), a cantina FFP-combiner wall-light draw (cantina interior).
The 32-bit gl11 Debug build is the v2.2-validated reference (left side of the A/B).
