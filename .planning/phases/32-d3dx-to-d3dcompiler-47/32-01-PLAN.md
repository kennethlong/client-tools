<objective>
Run the D-06 census/D3DAssemble probe — the CRITICAL EARLY DE-RISK GATE. This MUST complete before
the asm port commits (Wave 1). It refreshes the //hlsl vs //asm classification of the full TRE .vsh
corpus and empirically answers the one unprovable-from-static-analysis question: does `D3DAssemble`
(d3dcompiler_47) accept the SWG `.vsh` asm dialect AND produce bytecode that is SEMANTICALLY equivalent to
the `D3DXAssembleShader` baseline (size+hash, falling back to a disasm token-compare)?

Purpose: There is NO existence-proof that `D3DAssemble` accepts the SWG asm dialect (Restoration
precompiles bytecode and never calls it; the API is not even in the public d3dcompiler.h). The asm
half of this phase (D-03) is conditional on this gate. PASS / PASS-WITH-NOTE → Wave 1 ports the asm path to
D3DAssemble (32-03). FAIL-WITH-FOLLOWUP → the sanctioned D-03 fallback (keep `//asm` on
`D3DXAssembleShader` — note: the asm path is UNGUARDED today, see below — and schedule the
asm→D3DAssemble port as a must-land-before-Phase-33 follow-up).
Output: `32-CENSUS.md` (classification + raw dumpbin evidence + tri-state GATE VERDICT), `d3dassemble_probe.cpp` (the probe).

NOTE on the gate's tri-state shape (review fix #2): a bare PASS/FAIL is insufficient because Phase 32
delivers only PARTIAL SHADER-01 satisfaction (compile-path only, Win32 only), AND because benign
cross-assembler byte-variance must not force a false-FAIL. The FAIL outcome is named `FAIL-WITH-FOLLOWUP` —
it does NOT block the phase, but it DOES create a hard scheduling constraint (the asm→D3DAssemble port must
land before the Phase-33 x64 link, because D3DX is x64-hostile). The middle state `PASS-WITH-NOTE` covers
the "bytecode bytes differ but disasm tokens are identical" case (review fix #2 / Sonnet H1 / Opus).

NOTE on coverage (review fix #3 / Opus): this gate is PASS-ON-SAMPLE — it certifies the 3 baseline + the
Task-1 flag-listed constructs over the 16-extracted sample, NOT the full 96-//asm corpus. The recovery path
for a non-probed shader that fails render in 32-03 is defined in Task 3.
