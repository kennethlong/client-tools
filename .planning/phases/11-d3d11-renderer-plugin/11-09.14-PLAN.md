<risks>
1. **D3D9 sibling Stage construction quirks** â€” TA_invalid / TF_invalid fallback to TextureSampler defaults; global-texture short-circuit via `Direct3d11_TextureData::isGlobalTexture(tag)`. Mis-translating any of these silently produces wrong-but-non-crashing renders. Mitigation: read D3D9 sibling line-by-line (Direct3d9_StaticShaderData.cpp:297-378); zero-init the D3D11_SAMPLER_DESC; defensive log when invalid-fallback fires.

2. **TextureData pointer-to-pointer lifetime** â€” `Direct3d9_TextureData::const * const *m_texture` is a pointer to the engine `Texture`'s `m_graphicsData` field which can be re-assigned if the texture is reloaded. D3D9 sibling explicitly handles this via the double indirection (`*m_texture` at apply time, not at construct time). Plan 11-09.14 must mirror â€” store the pointer-to-pointer in `Stage::m_texture`, dereference only at apply.

3. **Sampler cache key collision risk** â€” `getOrCreateSampler(D3D11_SAMPLER_DESC const &)` keys via FNV-1a on the desc struct. Padding bytes vary across compiles â†’ key drift. Mitigation: zero-initialize the desc before fill (already in Plan 11-06 install path at StateCache.cpp:727).
