**Engine-side resource contract:** Each resource type has an engine-side handle class (`Texture`, `StaticVertexBuffer`, `DynamicVertexBuffer`, `StaticIndexBuffer`, `DynamicIndexBuffer`, `RenderTarget`) and a plugin-side data class. The plugin data class is constructed via the `Gl_api::create*` factory slot (engine passes the handle; plugin returns `*GraphicsData`). Engine then holds the data pointer for the resource's lifetime; deletes it via `delete` (data class has virtual destructor) when done.

void Direct3d11_StaticVertexBufferData::remove()
{
    delete ms_memoryBlockManager;
    ms_memoryBlockManager = NULL;
}

void *Direct3d11_StaticVertexBufferData::operator new(size_t size)
{
    UNREF(size);
    NOT_NULL(ms_memoryBlockManager);
    DEBUG_FATAL(size != sizeof(Direct3d11_StaticVertexBufferData), ("wrong new called"));
    return ms_memoryBlockManager->allocate();
}

void Direct3d11_StaticVertexBufferData::operator delete(void *memory)
{
    NOT_NULL(ms_memoryBlockManager);
    ms_memoryBlockManager->free(memory);
}
