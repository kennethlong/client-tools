### Phase 12 must-do scope (per convergent verdict)

1. **Asset PS pipeline** — PEXE bytecode is rejected by D3D11 `CreatePixelShader`; need either offline `.psh` → SM5 DXBC translation, or HLSL rewrite path mirroring the VS pipeline (Plan 11-05/06 era). This is THE structural blocker.
2. **Pass::apply constant uploads** — D3D9 uploads material, textureFactor (×2), textureScroll, stencil ref, fog mode, fullAmbient per pass; D3D11 currently uploads NONE in `Direct3d11_StaticShaderData::apply`. Even a perfect PS can't read constants never uploaded. **Cursor caught a quick-fix bug here**: `Direct3d11.cpp` `setPixelShaderUserConstants_impl` writes to a static `s_perMaterialShadow` but never calls `updatePS(2, &shadow, sizeof(shadow))` — pure dead-write.
3. **Stencil state mapping** — into `D3D11_DEPTH_STENCIL_DESC`. Stencil decals, outline passes, some interior effects need this.
4. **Gamma LUT post-process pass** — D3D11 equivalent of D3D9 `SetGammaRamp`. Renders resolved scene through a 1D/2D LUT encoding `pow(0.5 + contrast*(x*brightness - 0.5), 1/gamma)`.
5. **Optional FFP stage cascade generator** (CODEX's option e) — for FFP-only passes where no asset PS exists. Implement the dominant 10-12 of 26 `TextureOperation` enum modes; log+fallback the rest until live assets need them.

### Phase 12 should-do (after baseline)

- FFP lighting / material-source parity or shader-generated approximation
- Light manager constant parity vs D3D9 (dot3, extended lights)
- Bloom enable + alpha-fade/bloom packing
- Point-sprite emulation (stars, point particles)
- Resize / presentToWindow / cursor API completeness
- Backbuffer lock / writeImage parity

### Acceptable known limitations

- Dither ignored (no D3D11 equivalent)
- Exact D3D9 fullscreen gamma-ramp behavior replaced by swap-chain-local LUT
- Hardware point sprites rendered as 1px until explicit emulation
- Rare texture ops (BUMPENVMAP, DOTPRODUCT3, MULTIPLYADD) log+fallback until live assets surface them

### Hindsight architectural decisions to remember

1. **Dynamic PS keyed only by VS output** — correct as a linkage workaround, wrong as a material model. Must evolve to VS signature + pass material signature for Phase 12.
2. **Treating multi-stage SRV binding as visible progress by itself** — necessary plumbing but cannot affect pixels without generated PS stage evaluation.
3. **Defaulting unimplemented D3D9 states instead of logging pass usage** — made early smoke cleaner but hid which states were silently ignored.
4. **CODEX caveat preserved in Iter-44A**: D3D9 `Compare[]` table has `C_GreaterOrEqual` ↔ `C_NotEqual` swap vs enum comments. Mirror D3D9 for asset parity, don't "fix" it.
5. **Cursor correction in deep-dive**: `RibbonAppearance` does NOT bypass `setObjectToWorldTransformAndScale`. Identity-W is intentional for ParticleEmitter / SwooshAppearance / etc. (vertices already in world space).
