## TL;DR — Root cause found and fixed; awaiting smoke verification

After **27 iterations** chasing texture sampling, viewport data, SRV/RT conflicts, and shader linkage, CODEX and Cursor (independently, same data, same conclusion) identified the actual root cause: **`Direct3d11_StateCache::drawQuadList` was an unimplemented STUB**.

Every SWG UI textured quad flows through:
```
CuiManager::render() → UIManager::Render() → CuiLayerRenderer::flushRenderQueueQuads()
  → Graphics::drawQuadList() → Direct3d11_StateCache::drawQuadList() → no-op STUB → silent drop
```

The "engine renders only post-FX blit" symptom Iter-26 measured was literally true — every UI quad was dropped before reaching `applyPreDrawState`. All the prior debugging (texture content, alpha blending, SRV format, PS register linkage, etc.) was investigating the wrong draw call: the post-FX blit, which was the only thing reaching the pipeline.

**Iter-27 implements drawQuadList** using the Plan 11-09.15 fan-list IB pattern applied to quads.

## What's in the Iter-27 build (the actual fix)

### Code changes

**`Direct3d11_StateCache.cpp`** — implements `drawQuadList()`:

```cpp
void Direct3d11_StateCache::drawQuadList()
{
    int const vertCount = ms_currentVBVertexCount;
    if (vertCount < 4) return;
    int const numQuads = vertCount / 4;

    ++ms_drawQuadListCalls;
    if (numQuads > ms_quadListMaxQuads) ms_quadListMaxQuads = numQuads;

    if (!ensureQuadListIB(numQuads)) return;
    if (!applyPreDrawState(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST)) return;

    ID3D11DeviceContext * const ctx = Direct3d11_Device::getContext();
    ctx->IASetIndexBuffer(ms_quadListIB.Get(), DXGI_FORMAT_R16_UINT, 0);
    ctx->DrawIndexed(static_cast<UINT>(numQuads * 6), 0, 0);
    if (ms_currentIBValid && ms_currentIB)
        ctx->IASetIndexBuffer(ms_currentIB, ms_currentIBFormat, ms_currentIBOffset);
}
```

Plus `ensureQuadListIB(numQuads)` helper mirroring `ensureTriangleFanIB` — reusable IB with pattern `(4N+0, 4N+1, 4N+2, 4N+0, 4N+2, 4N+3)` per quad, doubling growth from initial capacity 256 quads (1024 verts), R16_UINT cap at 16383 quads (65532 verts).

**`Direct3d11_PixelShaderProgramData.cpp`** — REVERTED the per-VS PS UV-viz hack back to real `t.Sample(s, uv)`. The hack was producing the "gradient" Kenny was seeing — that was the blit's PS, not actual UI rendering. With drawQuadList now firing, the per-VS PS should sample real UI bitmap textures.
