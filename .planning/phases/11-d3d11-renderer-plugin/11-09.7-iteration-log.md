### Hypothesis

`Direct3d11.cpp:688` has `STUB(createVertexBufferVectorData);` and `:690` has `STUB(setVertexBufferVector);` — both were intentionally deferred at Plan 11-04 wiring time because no smoke had reached the skeletal-animation rendering path. Plan 11-09.6 unblocked shader compile, allowing the engine to proceed FURTHER, reaching this category.

Port `Direct3d9_VertexBufferVectorData` shape into a new `Direct3d11_VertexBufferVectorData` class. Wire both slots: `createVertexBufferVectorData` returns the new wrapper; `setVertexBufferVector` routes to `Direct3d11_StateCache::setVertexBufferVector` which calls `IASetVertexBuffers` across the multi-stream slot range + composes the input-layout cache key from the per-stream VBFormats.

### Investigation

D3D9 reference at `Direct3d9_VertexBufferVectorData.h`/`.cpp`:
- Class state: `IDirect3DVertexDeclaration9 *m_vertexDeclaration` (cached at construction time via `Direct3d9_VertexBufferVectorDescriptorMap::fetchVertexDeclaration`).
- Constructor: iterates VBVector's `m_vertexBufferList`, captures per-stream `VertexBufferFormat const *`, calls fetcher.
- Destructor: `m_vertexDeclaration->Release()`.
- Bind path at `Direct3d9.cpp:3764-3883`: iterates VBs, switches on type (T_static vs T_dynamic), per-stream `setStreamSource`, final `SetVertexDeclaration` from cached data.

D3D11 architectural difference: `ID3D11InputLayout` requires VS bytecode at creation time (D3D9's `IDirect3DVertexDeclaration9` is VS-agnostic). Plan 11-06 already implements a `(VBFormat, VS-bytecode-hash) -> ID3D11InputLayout *` cache via `Direct3d11_VertexDeclarationMap::getOrCreate`. Multi-stream case requires composing the cache key from per-stream VBFormats.

### Root cause

`STUB(createVertexBufferVectorData);` and `STUB(setVertexBufferVector);` at Direct3d11.cpp:688/690 — intentional deferral from Plan 11-02; surfaced as the next blocker when the skeletal-animation rendering path was first exercised in Plan 11-09.6's smoke.

Hypothesis confirmed.

### Fix

Per `11-09.7-PLAN.md` Tasks 1-3:

- **Task 1:** New class `Direct3d11_VertexBufferVectorData.{h,cpp}` — inherits `VertexBufferVectorGraphicsData`, holds per-stream `VertexBufferFormat const *` array (up to MAX_VERTEX_BUFFERS=2), accessor for the array consumed by `applyPreDrawState` input-layout cache lookup.
- **Task 2a:** `Direct3d11.cpp:688` STUB → `ms_glApi.createVertexBufferVectorData = createVertexBufferVectorData_impl`.
- **Task 2b:** `Direct3d11.cpp:690` STUB → `ms_glApi.setVertexBufferVector = Direct3d11_StateCache::setVertexBufferVector`.
- **Task 2c:** New `Direct3d11_StateCache::setVertexBufferVector(VertexBufferVector const &)` — iterates VB list, collects per-stream ID3D11Buffer + offset + stride, single batched `IASetVertexBuffers(0, count, buffers, strides, offsets)`.
- **Task 2d:** `applyPreDrawState` input-layout cache extended to compose key from per-stream VBFormats when a VBVector is the active geometry source.
- **Task 3:** `clientGraphics/VertexBufferVector.h:31` — add `friend class Direct3d11_VertexBufferVectorData;` (Rule-3 deviation; single-line; mirrors existing D3D9 friend at line 30).
