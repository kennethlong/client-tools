## Critical context / anti-patterns to carry forward

1. **Build slashes are PowerShell-only** — `Bash` mangles via Git Bash POSIX path conversion. Canonical command shape in earlier `.continue-here-archived-*` files.

2. **Background MSBuild exit codes can mislead through pipes** — file-redirect (`>`) and check `$LASTEXITCODE` explicitly. NEVER trust `Select-Object` tail's exit code.

3. **Debug log is APPEND mode** (Plan 11-08 Iter-1.7 file sink). When grepping for ERRORs after a smoke, FIND THE LATEST `=== D3D11 InfoQueue session begin ===` MARKER (`grep -n "session begin" stage/d3d11-debug.log | tail -1`) and filter `tail -n +<LINE>`. Otherwise you'll count ERRORs from prior smoke sessions. Cost-of-mistake: nearly mis-diagnosed Plan 11-09.8 as "TEXCOORD2 cascade not resolved" when it was 100% resolved.

4. **Don't touch Koogie's Direct3d9 working-tree mods.** Memory rule `feedback_dont_modify_koogie_changes.md`. Use explicit `git add` paths.

5. **rasterMajor values:** `5` (D3D9), `6` (FFP), `7` (VSPS), `11` (D3D11). NOT `9`. Memory: `project_rastermajor_values.md`.

6. **Just do reversible local changes** — config edits, stage/ toggles, file flips, push-after-land. Don't wait for Kenny. Memory: `feedback_just_do_reversible_local_changes.md`.

7. **stage/client_d.cfg deferred cleanup** — accumulated Phase 11 test settings; clean up AFTER visual parity completes. Memory: `project_stage_client_d_cfg_cleanup_todo.md`.
