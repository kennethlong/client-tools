<objective>
Defensive audit of the 40+ MB of debug-layer traffic accumulated during the Plan 11-09.11 close smoke (stage/d3d11-debug.log grew 156→198 MB in a single session). Two parallel goals:

1. **Cleanliness sweep:** verify no silent-degradation bugs are hiding behind the all-magenta rendering. Past plans (notably Plan 11-09.7→11-09.8) showed that ERROR / WARNING counters can quietly accumulate while the engine remains nominally running -- the visible FATAL cascade only catches the catastrophic cases. With FATALs cleared, the residual silent-error rate becomes load-bearing.

2. **PEXE-rejection evidence collection:** Kenny smoke confirmation: "all I saw was magenta triangles, just want to make sure you know, no real terrain was rendered." That means **every** real-PS asset shader is failing to compile / bind, and Plan 11-09 Iter-2's magenta fallback PS is covering 100% of draws. The Plan 11-05 PEXE rejection mechanism (asset shader bytecode FATALs at compile time) is the load-bearing root cause of the all-magenta state. The debug-layer should show evidence: CreateShader failures, PSSetShader-with-null bindings, or similar. That evidence becomes the entry point for Plan 11-11 visual parity.

Approach:
- Find the LATEST `=== D3D11 InfoQueue session begin ===` marker in `stage/d3d11-debug.log` (file is append-mode per Plan 11-08 Iter-1.7; the marker delimits sessions). Anything before that marker is prior-session noise; we filter `tail -n +<LINE>`.
- Extract `[ERROR]` and `[WARNING]` lines from the post-marker range. Count by D3D11 debug-layer message-id (e.g. `MESSAGE_ID_BIND_INPUTLAYOUT_NOT_SET`, `INVALIDARG_RETURN`, etc.) -- the InfoQueue format prefixes each message with its category + id.
- For each distinct category, triage with one of 4 verdicts:
  - **real-bug-fix-now** -- concrete D3D11 plugin defect; Iter-1 mini fix lands.
  - **benign-by-design** -- expected debug-layer noise (e.g. shader-bytecode-fingerprint mismatch warnings that fire by D3D11 design when reusing input layouts cross-shader).
  - **Plan-11-11-territory** -- visual-parity / PEXE-rejection fallout; capture for the next plan's intake.
  - **Plan-11-12+-territory** -- defer to native-PIX / advanced-features territory.
- Write findings + verdicts to `11-09.12-AUDIT.md`.
- If `real-bug-fix-now` verdicts surface, apply Iter-1 fix(es) and rebuild + re-smoke.

Scope ceiling: ONE iteration. The audit pass itself is investigative (no code change). Any concrete fixes triggered by the audit are bounded by D-05 (D3D11 plugin only).
</objective>

### Task 1 — Locate the latest session boundary in the debug log

**1a.** `wc -l stage/d3d11-debug.log` to capture total line count for baseline.

**1c.** Compute the post-marker range: `tail -n +<LATEST_LINE_NUMBER>`. Note the line count.

### Task 2 — Extract + categorize session-only ERROR / WARNING traffic

**2a.** `tail -n +<LATEST_LINE_NUMBER> stage/d3d11-debug.log | grep -E "\[ERROR\]|\[WARNING\]" | wc -l` -- total session-only count.

**2b.** Extract distinct message-id categories via `... | grep -oE "MESSAGE_ID[A-Z0-9_]+" | sort | uniq -c | sort -rn` (or equivalent: the InfoQueue format prefixes each line with `[CATEGORY] [SEVERITY] [STATE_CREATION_..._id]` -- exact regex per the format observed in earlier logs).

**2c.** For the top 10 categories by count, extract 1-3 sample lines per category to capture the message body (helps triage benign-by-design vs real-bug).

### Task 3 — Triage + write 11-09.12-AUDIT.md
