## Iter-1: Reflection + augment + phantom slot 15 bind

**Date:** 2026-05-21 (late evening, post-Plan-11-09.7 close `ef94b6662`)

### Symptom (carried from Plan 11-09.7 close smoke)

20+ identical D3D11 debug-layer ERRORs in `stage/d3d11-debug.log`:
```
ID3D11Device::CreateInputLayout: The provided input signature expects to read an
element with SemanticName/Index: 'TEXCOORD'/2, but the declaration doesn't
provide a matching name.
```

Engine VS bytecode (compiled under Plan 11-09.6 `vs_4_0`) declares TEXCOORD2 input even when unused. Engine VBFormat for many meshes doesn't provide TEXCOORD2. SM4 strict signature validation rejects layout → CreateInputLayout returns FAIL → cache stores nullptr → applyPreDrawState skips draws → no geometry on those passes.

Partial-magenta draws visible for the subset of VS/VBFormat combinations that match. Engine progressed deep into char-creation (login + avatar composition + hair_s40 mesh load) before access violation.

### Hypothesis

Reflect each VS bytecode's input signature (`D3DReflect` + `ID3D11ShaderReflection::GetInputParameterDesc`); augment the input layout with phantom zero-buffer elements at slot 15 for any non-system semantics the VBFormat-derived elements don't cover; bind a 16-byte zero-filled ID3D11Buffer at slot 15 with stride=0 before every draw. CreateInputLayout validation passes; failed draws now succeed (geometry visible as Plan 11-09 fallback-PS magenta).

CODEX pre-implementation consult (`11-09.8-CODEX-CONSULT-texcoord2-signature-mismatch.md`):
- Confirmed expected SM4 behavior (vs_4_0 preserves declared inputs in signature).
- Ranked Bucket C-lite/A2 hybrid as recommended.
- Specified slot 15 (NOT slot 7) -- engine uses low slots for real streams.
- Skip SystemValueType != D_NAME_UNDEFINED (SV_VertexID etc.).
- c0000005 should NOT be assumed to share root cause; fix layouts first.

### Investigation

CODEX consult (`11-09.8-CODEX-CONSULT-texcoord2-signature-mismatch.md`) provided design verdict: C-lite/A2 hybrid (reflection + phantom zero buffer at slot 15). Build attempts:
- Iter-1-01 EXIT=1 — `Direct3d11_ReflectedVSInput` was forward-declared in `Direct3d11_VertexBufferDescriptorMap.h` but not defined in the .cpp → C2027 "use of undefined type". Fix: include `Direct3d11_VertexShaderData.h` (full def) + `<cstring>` (for `_stricmp`) in `Direct3d11_VertexBufferDescriptorMap.cpp`.
- Iter-1-02 EXIT=0 — 0 errors, 0 warnings.

### Root cause

CODEX-confirmed: expected SM4 behavior. `vs_4_0` profile preserves declared inputs in the bytecode signature even when unused (D3D9-era HLSL routinely declares TEXCOORD0..N inputs). Engine VBFormats can't fabricate missing semantics. `CreateInputLayout` validation requires the layout cover the signature. Fix: reflect once-per-VS, augment layouts with phantom elements pointing at a zero-stride buffer in slot 15.

Hypothesis confirmed verbatim.

### Fix

Per `11-09.8-PLAN.md` Tasks 1-5:

- **Task 1:** D3DReflect in Direct3d11_VertexShaderData::compile post-D3DCompile. Compact `ReflectedInput` struct. Filter SystemValueType. Accessor.
- **Task 2:** 16-byte zero-filled BIND_VERTEX_BUFFER in Direct3d11_Device::create. getPhantomZeroBuffer() accessor.
- **Task 3:** Direct3d11_VertexBufferDescriptorMap::augmentWithPhantomElements() appends missing non-SV semantics at InputSlot=15, AlignedByteOffset=0. Format derived from reflected ComponentType+Mask.
- **Task 4:** VertexDeclarationMap::getOrCreate + getOrCreateMultiStream call augment before CreateInputLayout. Cache key unchanged (augmentation deterministic).
- **Task 5:** applyPreDrawState binds phantom buffer at slot 15 unconditionally before every draw.
