### CODEX verdict (2026-05-22)

**Bucket A confirmed**, with two corrections:

1. **Select by `m_textureIndex == 0`, not "first iterator."** D3D9 indexes `m_stage[(*i)->m_textureIndex]` (Direct3d9_StaticShaderData.cpp:789); the dynamic fallback PS samples `t0`/`s0` so slot 0 means texture-index 0. If no `TextureSampler` in a pass has `m_textureIndex == 0`, unbind SRV0 — fallback-to-first is diagnostic experiment, not correct architecture.

2. **Route through public `Direct3d11_StateCache` setters, not direct `PSSet*` calls.** Preserve the Plan 11-06 lazy-bind model (single `PSSetShaderResources(0, kMaxSRVs, ms_boundSRV)` flush per draw at `applyPreDrawState`).

Plus: the Iter-4 `sampler0=NULL` diagnostic line was **misleading** — `PSGetSamplers` inside `selectFallbackPSForVS` reads device state BEFORE `applyPreDrawState`'s late `PSSetSamplers` flush, so it sees stale state from the previous draw. Default samplers ARE pre-filled at install (`StateCache.cpp:755-756`); they just hadn't been applied yet at the diagnostic site. The real blocker was always SRV0.

CODEX Q3..Q5 specifics:
- **Anisotropy:** clamp `textureData.maxAnisotropy` to `[1,16]` per MS docs — no `getMaxAnisotropy()` caps query.
- **Filter mapper:** `TF_anisotropic` → `D3D11_FILTER_ANISOTROPIC` + clamped aniso; `TF_linear`/`TF_point` direct; `TF_invalid`/cubic/unsafe → linear fallback; `TF_none` on mip → point mip.
- **Unbind on absence:** when no `TextureSampler` has `m_textureIndex == 0`, set `ms_boundSRV[0] = nullptr` (avoid sticky cross-shader bleed). Sampler0 default is fine.
- **No material work needed.** Dynamic fallback PS reads only `t.Sample(s, input._v0.xy)`.

### Investigation + fix

**Task 1: Public StateCache setters.** Added three to `Direct3d11_StateCache`:
- `setPixelShaderResource(int slot, ID3D11ShaderResourceView *srv)` — writes `ms_boundSRV[slot]` (null is valid for unbind).
- `setPixelShaderSampler(int slot, D3D11_SAMPLER_DESC const &desc)` — resolves desc through existing `getOrCreateSampler` (FNV-1a-keyed cache from Plan 11-06); writes `ms_boundSampler[slot]`. Null resolution falls back to `ms_defaultSampler.Get()` (CODEX correction #2 safety: default sampler is always pre-filled at install).
- `setPixelShaderSampler(int slot, ID3D11SamplerState *sampler)` — overload taking the resolved object.

All three bounds-check `slot` against `kMaxSRVs`/`kMaxSamplers` (16 each).

**Task 2: Per-pass Stage0 extraction in `Direct3d11_StaticShaderData::construct()`.**

Added `struct Stage0 { bool m_present; Direct3d11_TextureData const * const *m_texture; D3D11_SAMPLER_DESC m_samplerDesc; }` on the class, stored as `std::vector<Stage0> m_passStage0` parallel to `m_passVS`/`m_passPS`.

In `construct()`, after the VS/PS extraction loop body, iterate `pass->m_pixelShader->m_textureSamplers`. For each:
- Skip unless `(*i)->m_textureIndex == 0` (CODEX correction #1).
- Mark `stage.m_present = true`.
- Pre-fill `TextureData` with `TA_wrap`/`TF_linear`/aniso=1/texture=NULL defaults. If `m_textureTag != 0`, call `shader.getTextureData(tag, td)` — overrides on success, leaves defaults on failure.
- Global-texture short-circuit via `Direct3d11_TextureData::isGlobalTexture(tag)` → `getGlobalTexture(tag)` returns the pointer-to-pointer.
- Else if `textureData.texture` resolved → `reinterpret_cast<Direct3d11_TextureData const * const *>(textureData.texture->getGraphicsDataAddress())`.
- Build `D3D11_SAMPLER_DESC` via file-local helper `buildSamplerDescForStage0` (CODEX Q3 mapping rules).
- `break` after slot-0 match (Bucket A scope; don't continue iterating).

Helper `toD3D11Filter(min, mag, mip)` returns one of the eight `D3D11_FILTER_MIN_*_MAG_*_MIP_*` permutations or `D3D11_FILTER_ANISOTROPIC` per CODEX Q3 rules. Helper `toD3D11Address` maps the 5-value `TextureAddress` enum.

**Task 3: Binding in `Direct3d11_StaticShaderData::apply()`.**

Inside the existing `!cacheHit` branch, after the VS/PS pointer set:
- `!stage.m_present` → `setPixelShaderResource(0, nullptr)` (CODEX Q4 unbind-on-absence).
- `stage.m_present && (!stage.m_texture || !*stage.m_texture)` → `setPixelShaderResource(0, nullptr)` + `setPixelShaderSampler(0, stage.m_samplerDesc)`. Sampler bound for defined slot state; SRV unbound → fallback-PS reads 0 → magenta/black correctly.
- Resolved texture → `setPixelShaderResource(0, (*stage.m_texture)->getShaderResourceView())` + `setPixelShaderSampler(0, stage.m_samplerDesc)`.

Out-of-bounds pass index → `setPixelShaderResource(0, nullptr)` alongside the existing VS/PS clear.

**Task 4: Diagnostics rework.** Removed the misleading Iter-4 first-dynamic-PS `sampler0=NULL` diagnostic from `Direct3d11_StateCache::selectFallbackPSForVS` (CODEX correction #2: it read stale device state). Replaced with lifetime counters:
- `s_passesTotal`, `s_passesWithSampler0`, `s_passesWithSampler0TextureResolved` — file-static in `Direct3d11_StaticShaderData.cpp`; incremented in construct(); logged in `Direct3d11_StaticShaderData::remove()` shutdown.
- `ms_drawsWithSRV0Bound` — namespace static in `Direct3d11_StateCache.cpp`; incremented inside `applyPreDrawState`'s flush block when `ms_boundSRV[0] != nullptr`; logged in `Direct3d11_StateCache::remove()` shutdown.

Cross-reference comment in `Direct3d11_PixelShaderProgramData.h:96-101` updated from "CODEX-flagged stale premise persists" → historical note acknowledging the gap is closed.
