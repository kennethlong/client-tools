### CODEX verdict (2026-05-22)

**Bucket A confirmed** with three important corrections:

1. **R16_UINT, not R32_UINT.** Engine `Index` is 16-bit; `Direct3d11_StateCache.cpp:1196` already assumes `DXGI_FORMAT_R16_UINT`; realistic SWG fans tiny. Hard guard for `requestedVertices > 65535` (`DEBUG_FATAL` in debug + log/skip in release). Initial capacity 1024; doubling growth; capped at 65535.

2. **Indexed-fan FIX-NOW path was wrong.** The consult proposed "CPU-readback engine IB at draw time" — CODEX corrected: `Direct3d11_StaticIndexBufferData::lock(readOnly)` allocates a zeroed CPU buffer; it does NOT actually read engine indices back (`Direct3d11_StaticIndexBufferData.cpp:96`). Dynamic IB also has no persistent CPU shadow. Real fix would need CPU-side index shadows added at lock/unlock — too much for 11-09.15. **DEFER + INSTRUMENT only.** Mandatory caveat: `ShaderPrimitiveSetTemplate.cpp:673` CAN dispatch `SPSPT_indexedTriangleFan` from asset data, so source search doesn't prove indexed fans unused; counters + one-shot DEBUG_REPORT_LOG_PRINT tell smoke whether Plan 11-09.16 is required.

3. **Do NOT mutate `ms_currentIB*`.** Bind fan IB after `applyPreDrawState`, leave the StateCache shadow alone. Next indexed draw rebinds via existing guard at `cpp:601`. (CODEX also corrected my "PSGetIndexBuffer" typo — the correct API is `IAGetIndexBuffer`; avoid it.)

### Investigation + fix

**Task 1: Reusable fan-to-list IB infrastructure.** Added 6 namespace statics + 2 constants to `Direct3d11_StateCacheNamespace`:
```
constexpr int kTriangleFanIBInitialCapacityVerts = 1024;
constexpr int kTriangleFanIBMaxCapacityVerts     = 65535;
ComPtr<ID3D11Buffer> ms_triangleFanIB;
int ms_triangleFanIBCapacityVerts = 0;
int ms_drawTriangleFanCalls               = 0;
int ms_drawPartialTriangleFanCalls        = 0;
int ms_triangleFanIBRebuilds              = 0;
int ms_triangleFanMaxVertices             = 0;
int ms_drawIndexedTriangleFanCalls        = 0;
int ms_drawPartialIndexedTriangleFanCalls = 0;
```

Helper `ensureTriangleFanIB(vertCount)`:
- DEBUG_FATAL (release: log/skip) if `vertCount > 65535`.
- Cache hit if `ms_triangleFanIB && vertCount <= ms_triangleFanIBCapacityVerts`.
- Otherwise rebuild: doubling growth from current capacity (or initial 1024 if none) up to `vertCount`, capped at 65535. CPU staging `std::vector<uint16_t>` filled with `(0, i, i+1)` pattern; `CreateBuffer` with `USAGE_IMMUTABLE` / `BIND_INDEX_BUFFER` / `D3D11_SUBRESOURCE_DATA`. On failure: clear, log, return false (next draw retries).
- On success: assign to ComPtr (auto-releases previous); increment `ms_triangleFanIBRebuilds`.

`install()` pre-allocates at `kTriangleFanIBInitialCapacityVerts` (1024) — avoids first-draw stall during world load. `remove()` releases ComPtr.

**Task 2: drawTriangleFan + drawPartialTriangleFan reimplementation.**

For `drawTriangleFan()`:
- Compute `triCount = vertCount - 2`.
- Increment `ms_drawTriangleFanCalls` + update `ms_triangleFanMaxVertices`.
- `ensureTriangleFanIB(vertCount)`; skip on failure.
- `applyPreDrawState(TRIANGLELIST)`.
- Bind fan IB via direct `IASetIndexBuffer` (after applyPreDrawState; do NOT mutate `ms_currentIB*` per CODEX Q4).
- `DrawIndexed(triCount * 3, 0, 0)`.

For `drawPartialTriangleFan(startVertex, primitiveCount)`:
- Same shape, but `vertCount = primitiveCount + 2` and `DrawIndexed(primitiveCount * 3, 0, startVertex)`. `BaseVertexLocation = startVertex` shifts the static (0,i,i+1) pattern automatically.

For `drawIndexedTriangleFan` + `drawPartialIndexedTriangleFan` (DEFER + INSTRUMENT per CODEX Q3):
- Keep TRIANGLELIST topology behavior (still broken; no worse than before).
- Increment per-call counter.
- Per-function `static bool s_warnedOnce` guard with DEBUG_REPORT_LOG_PRINT identifying the broken function + Plan 11-09.16 pointer.
