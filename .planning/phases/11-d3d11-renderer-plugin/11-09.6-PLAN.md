<objective>
Remove the SM2.x-style 256-instruction-slot cap that's blocking world-entry under the D3D11 plugin. The cap was being enforced by Plan 11-07 Iter-3's choice of `vs_4_0_level_9_3` compile profile (chosen specifically to dodge vs_5_0's keyword reservation problem; the level_9_3 trade-off inherits SM2.x feature limits). Plan 11-09.6 bumps VS profile to `vs_4_0` (PS helper to `ps_4_0` for symmetry) — keeps the D3D9-era HLSL syntax compatibility while dropping the instruction-slot ceiling.

**Surface symptom:** D3DCompile error X5615 on `vertex_program/a_specmap_bump_vs20_for_ps20.vsh` (293 instructions vs 256 max for vs_2_x). Reproducible on every char-create / char-select session post-Plan-11-09.5; crash dump preserved at `stage/SwgClient_d.exe-unknown.0-20260521202203.{mdmp,txt}` (Plan 11-09.5 D3D11 smoke).

**Predicted cascade (CODEX consult Q2 in priority order):** legacy SM4 syntax/keyword fallout → register/constant layout fallout → VS output / PS input runtime mismatch → runtime constant-binding mismatch. PS-era constructs (texkill / texldp / sampler+texture split) lower-risk because stock PEXE rejected pre-compile.

**Scope ceiling:** ONE iteration unless the post-fix smoke surfaces an immediate cascade-error. The fix is 4 line changes; cascade-discovery iterations open as Plan 11-09.6 Iter-2+ following the Plan 11-07 / 11-09 fix-by-fix pattern.

Full pre-implementation CODEX consult preserved at `.planning/phases/11-d3d11-renderer-plugin/11-09.6-CODEX-CONSULT-shader-target-bump.md`.
</objective>
