## Iter-1: Port updateWindowSettings + wire into install + replace setWindowedMode STUB

**Date:** 2026-05-21 (afternoon session, post-Plan-11-09 close at `077b133d6`)

### Symptom (carried from Plan 11-09 close)

Plan 11-09 closed 2026-05-21 at visible-magenta milestone (commits `f3b27054b` + `077b133d6`). Smoke harness validated by running `stage/SwgClient_d.exe` and then executing `tools/d3d11-smoke/show-window.ps1` to force `ShowWindow(SW_SHOW) + SetWindowPos + SetForegroundWindow`. Without the script: window never appears; user sees nothing.

Script preamble (`tools/d3d11-smoke/show-window.ps1` lines 2-5): *"Originated in Phase 11 Plan 11-07 Iter-17, where the engine's normal `ShowWindow(SW_SHOW)` path didn't fire on launch. Plan 11-08+ smokes carry the workaround forward until the engine-init path is investigated."*

### Hypothesis

The D3D11 plugin is missing the equivalent of `Direct3d9::updateWindowSettings()` -- the function the D3D9 plugin uses to reveal the engine-created hidden window via `SetWindowPos(... SWP_SHOWWINDOW)`. Diagnosis verified during session 2026-05-21:

1. `Os::install` (Os.cpp:142) calls `CreateWindow(... WS_POPUP, ..., NULL)` -- no `WS_VISIBLE`; no `ShowWindow`. Engine creates window hidden by design.
2. `Direct3d9::install` (Direct3d9.cpp:1535) calls `updateWindowSettings()` which lands `SetWindowPos(ms_window, HWND_NOTOPMOST, x, y, w, h, SWP_NOCOPYBITS | SWP_SHOWWINDOW)`.
3. `Direct3d11::install` (Direct3d11.cpp:460) has no equivalent; `grep -rn "updateWindowSettings\|SetWindowPos\|SWP_SHOWWINDOW\|ShowWindow" src/engine/client/application/Direct3d11/` returns **0 matches** as of HEAD `077b133d6`.
4. `Direct3d11.cpp:583` explicitly stubs `setWindowedMode` to scaffold_fatal_stub.

Expected post-fix: window appears on its own; show-window.ps1 becomes a redundant diagnostic helper.

### Investigation

Source-tree audit confirmed hypothesis:

1. `grep -rn "ShowWindow\|CreateWindowEx" src/engine/` â€” `Os.cpp:142` calls `CreateWindow(... WS_POPUP, 0, 0, 640, 480, NULL, NULL, instance, NULL)`. No `WS_VISIBLE`. No `ShowWindow` anywhere in `Os::install` or `Os::installCommon`. Engine creates the application window hidden by design (game-launch path; not the editor-tool path).
2. `grep -n "SetWindowPos\|SWP_SHOWWINDOW" Direct3d9.cpp` â€” `updateWindowSettings()` at `Direct3d9.cpp:1870-1930`, called from `Direct3d9.cpp:1535` during `Direct3d9::install` (right after the device-create loop succeeds). Terminates in `SetWindowPos(ms_window, HWND_NOTOPMOST, ..., SWP_NOCOPYBITS | SWP_SHOWWINDOW)` at line 1913 (windowed) / 1927 (fullscreen). The `SWP_SHOWWINDOW` flag is what reveals the window.
3. `grep -rn "updateWindowSettings\|SetWindowPos\|SWP_SHOWWINDOW\|ShowWindow" src/engine/client/application/Direct3d11/` â€” **0 matches** as of HEAD `077b133d6`. The D3D11 plugin has no window-show step.
4. `Direct3d11.cpp:583` had `STUB(setWindowedMode);` â€” engine's setWindowedMode slot routed to scaffold_fatal_stub.
5. State already plumbed: `Direct3d11_Device.cpp:51-54` declares `ms_window`, `ms_width`, `ms_height`, `ms_windowed`. `Gl_install` struct (Gl_dll.def:68) provides `engineOwnsWindow` bool. DXGI `IDXGISwapChain1::GetContainingOutput()` returns `IDXGIOutput*` whose `GetDesc().DesktopCoordinates` substitutes for D3D9's `IDirect3D9::GetAdapterMonitor + Win32 GetMonitorInfo` chain.

### Root cause

D3D11 plugin missing the equivalent of `Direct3d9Namespace::updateWindowSettings()`. Engine creates window hidden via `Os::install` (Os.cpp:142, `WS_POPUP` without `WS_VISIBLE`, no `ShowWindow`); render plugin is the established place to reveal it after device/swapchain creation. D3D9 plugin owns this; D3D11 plugin missed it during the original Plan 11-03 scaffold and the gap was papered over by `tools/d3d11-smoke/show-window.ps1` from Plan 11-07 Iter-17 onward.

Hypothesis confirmed verbatim. No surprises during investigation.

### Fix

Per `11-09.5-PLAN.md` Task 1:

- **1a.** Add `ms_windowX = INT_MAX`, `ms_windowY = INT_MAX`, `ms_borderlessWindow = false` to `Direct3d11_Device.cpp` anonymous namespace.
- **1b.** Author `Direct3d11Namespace::updateWindowSettings()` in `Direct3d11.cpp`. Verbatim mirror of `Direct3d9Namespace::updateWindowSettings()` with these substitutions:
  - `ms_engineOwnsWindow` -> `Direct3d11_Device::engineOwnsWindow()` (add accessor if missing).
  - `ms_window/width/height/windowed` -> `Direct3d11_Device::getWindow()/getWidth()/getHeight()/isWindowed()` (add `isWindowed()` if missing).
  - `ms_direct3d->GetAdapterMonitor(ms_adapter)` -> DXGI: `swapChain->GetContainingOutput()` then `IDXGIOutput::GetDesc().DesktopCoordinates`. Fallback if null: `MonitorFromWindow(ms_window, MONITOR_DEFAULTTOPRIMARY) + GetMonitorInfo`.
  - Final SetWindowPos preserved exactly: `SWP_NOCOPYBITS | SWP_SHOWWINDOW`.
- **1c.** Call `updateWindowSettings()` from `Direct3d11::install` after `Direct3d11_Device::create` returns true (~line 478).
- **1d.** Replace `STUB(setWindowedMode);` (line 583) with real binding. Minimum-viable: handle boot-time `windowed=true` no-op; FATAL on fullscreen toggle attempts pending future plan if DXGI SetFullscreenState surfaces complications.
- **1e.** D-05 protection check after every edit.
- **1f.** PowerShell-only MSBuild on all 5 targets.
