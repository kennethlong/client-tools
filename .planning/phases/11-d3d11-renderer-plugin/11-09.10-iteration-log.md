## Iter-1: Inline no-op functions for 6 point-sprite slots

**Date:** 2026-05-21 (late evening, post-Plan-11-09.9 close `39d340a40`)

### Symptom (carried from Plan 11-09.9 close smoke)

```
FATAL c57b5047: Direct3d11 plugin: scaffold-only -- unimplemented Gl_api slot called (Plan 11-02 expected; Wave 3+ replaces this)
```

Call chain:
- `StarAppearance::LocalShaderPrimitive::draw` (StarAppearance.cpp:337) — `Graphics::setPointSize(2.f)` for night-sky star rendering
- `Graphics::setPointSize` (Graphics.cpp:832) — `NOT_NULL(ms_api->setPointSize); ms_api->setPointSize(size);`
- `ms_api->setPointSize` → Plan 11-02 STUB at Direct3d11.cpp:617 → `scaffold_fatal_stub` → FATAL.

Crash dump preserved: `stage/SwgClient_d.exe-unknown.0-20260522032042.{mdmp,txt}` (UpTime 50 / MainLoop 2885; Terrain=`terrain/tatooine.trn`, Player at Tatooine in-world coords — proves world load completed and scene render began).

### Hypothesis

Direct3d11.cpp:613-616 already says the design intent is "no-op these in D3D11 because there's no fixed-function point-sprite control." The STUB macro at Direct3d11.cpp:594 is `ms_glApi.slot = reinterpret_cast<decltype(...)>(scaffold_fatal_stub)` — it routes to FATAL, not to a no-op. The fix is to define 6 real no-op impl functions and wire them to the slots, replacing the 6 STUB(...) lines.

Gl_dll.def:129-134 signatures (all return void):
- `setPointSize(real size)`
- `setPointSizeMax(real size)`
- `setPointSizeMin(real size)`
- `setPointScaleEnable(bool bEnable)`
- `setPointScaleFactor(real A, real B, real C)`
- `setPointSpriteEnable(bool bEnable)`

`Direct3d11_StateCache::drawPointList` is already wired (Direct3d11.cpp:722 → Direct3d11_StateCache.cpp:1206); POINTLIST topology renders as 1-pixel hardware-default points without state-cache work. The state setters are pure pass-throughs whose values would feed a fixed-function pipeline that doesn't exist in D3D11.

### Investigation

Confirmed the precedent for `_impl` no-op functions lives in `namespace Direct3d11Namespace` at file scope (Direct3d11.cpp:67, ~246-264). `setBrightnessContrastGamma_impl(float,float,float)` at line 246 + `update_impl(float)` at line 251 are the canonical pattern: file-scope free function inside the plugin namespace, empty body, slot-pointer wired via `ms_glApi.setBrightnessContrastGamma = setBrightnessContrastGamma_impl;` in install().

Confirmed Gl_dll.def:129-134 declares the 6 slots with parameter type `real` (FirstSharedFoundation.h:46 `typedef float real;` — a float alias). Matched the slot signatures directly using `real` rather than reduxing to `float` (compiles identically; the `real` typedef is the Gl_api contract spelling).

Confirmed `Direct3d11_StateCache::drawPointList` is already wired at Direct3d11.cpp:722 → Direct3d11_StateCache.cpp:1206 (POINTLIST topology + applyPreDrawState + Draw). No work needed on the draw side; the state setters are the only gap.

### Root cause

Confirms hypothesis verbatim. The STUB macro at Direct3d11.cpp:594 is `reinterpret_cast<decltype(...)>(scaffold_fatal_stub)` — it FATALs on entry rather than no-oping. The comment block at lines 613-616 declared the intent ("D3D11 has no fixed-function point-sprite control; Plan 11-06 stubs out the setters as no-ops") but the mechanism was wrong: STUB → FATAL instead of a real no-op slot wire-up. Plan 11-09.10 realizes the intent by introducing 6 real `_impl` no-op functions and replacing the STUB calls with direct slot assignments.

### Fix

Per `11-09.10-PLAN.md` Tasks 1-2:

- **Task 2a:** 6 new file-scope `_impl` functions added to `namespace Direct3d11Namespace` in Direct3d11.cpp after `update_impl` (around line 256). Empty bodies; param names elided via `/*comment*/`. Preceded by a multi-line comment block explaining why they no-op (D3D11 has no fixed-function point control; SV_POINTSIZE is FL9.x-only; we run vs_4_0 / FL10+) and what the visible fidelity trade is (1-pixel hardware-default stars; Plan 11-11 visual-parity decides whether to emulate via GS quad expansion or CPU expansion to TRIANGLELIST).

- **Task 2b:** Replaced the 6 STUB(...) lines at Direct3d11.cpp:617-622 with `ms_glApi.setPoint... = setPoint..._impl;` assignments. Slot names mirror the Gl_dll.def declarations exactly.

- **Task 2c:** Updated the preceding comment block to reflect Plan 11-09.10's intent realization and cross-reference Plan 11-09.9 close smoke as the trigger.
