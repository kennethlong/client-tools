<objective>
Fix the half-screen render artifact Kenny surfaced in the Plan 11-09.14 close smoke. `Direct3d11_StateCache::drawTriangleFan` and its three sibling variants currently submit triangle-FAN vertex data with `D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST` topology â€” D3D11 has no native fan topology and the Plan 11-06 STUB never expanded the conversion. For a canonical 4-vert fullscreen-quad fan (SkyBox6Sided face, sun, moon, planet, sprite, beam, marker, etc.):

- **As fan:** triangles `(0,1,2)` + `(0,2,3)` â†’ full quad (2 triangles)
- **As list (current STUB behavior):** triangle `(0,1,2)` only â†’ upper-right half (TL/TR/BR); vertex 3 (BL) discarded

This produced "magenta upper-right triangle" pre-Plan-11-09.14 (everything was fallback-magenta) and "textured upper-right triangle / black or background lower-left" post-Plan-11-09.14 once SRV0 binding wired real textures to the visible half.

**CODEX verdict (2026-05-22):** Bucket A confirmed â€” reusable static fan-to-list IB at install time (mirrors D3D9 plugin's `drawQuadList` pattern at `Direct3d9.cpp:2380` / `:4277`). Three important corrections to the working hypothesis:

1. **R16_UINT, not R32.** Engine `Index` is 16-bit; `Direct3d11_StateCache.cpp:1196` already assumes `DXGI_FORMAT_R16_UINT`. Hard guard for `requestedVertices > 65535` (log/skip; fatal in debug). Initial capacity 1024; doubling growth; capped at 65535.

2. **Indexed-fan FIX-NOW path was wrong.** My consult proposed "CPU-readback the engine's IB at draw time" â€” but `Direct3d11_StaticIndexBufferData::lock(readOnly)` allocates a zeroed CPU buffer; it does NOT read engine indices back (`Direct3d11_StaticIndexBufferData.cpp:96`). Dynamic IB also has no persistent CPU shadow. A real indexed-fan fix would require ADDING CPU-side index shadows in the D3D11 IB data classes at lock/unlock time â†’ too much for 11-09.15. **DEFER + INSTRUMENT only.** Mandatory caveat: source search does NOT prove indexed fans unused; `ShaderPrimitiveSetTemplate` can dispatch `SPSPT_indexedTriangleFan` from asset data at `clientGraphics/src/shared/ShaderPrimitiveSetTemplate.cpp:673`. The one-shot DEBUG_REPORT_LOG_PRINT diagnostic + counters tell us whether Plan 11-09.16 is required.

3. **Do NOT save/restore device IB state. Do NOT mutate `ms_currentIB*`.** Bind fan IB directly after `applyPreDrawState`, issue `DrawIndexed`, leave the StateCache shadow alone. Next indexed draw rebinds `ms_currentIB` via the existing guard at `Direct3d11_StateCache.cpp:601` (`if (ms_currentIBValid && ms_currentIB)`). If `ms_currentIBValid` is false before an indexed draw, that draw was already invalid by engine protocol â€” not our problem. Avoid `IAGetIndexBuffer` (CODEX corrected my "PSGetIndexBuffer" typo too).

### Task 1 â€” Reusable triangle-fan index buffer + ensure helper

**1a.** Add namespace statics to `Direct3d11_StateCacheNamespace`:
- `ComPtr<ID3D11Buffer> ms_triangleFanIB` â€” the reusable expanded IB.
- `int ms_triangleFanIBCapacityVerts` â€” max fan-vertex-count the current IB can serve (so `(cap-2)*3` 16-bit indices land inside the buffer).
- Initial capacity constant `kTriangleFanIBInitialCapacityVerts = 1024` (per CODEX Q2).
- Hard cap constant `kTriangleFanIBMaxCapacityVerts = 65535` (per CODEX Q2 â€” R16_UINT limit; fans approaching this size are pathological).

**1b.** Helper `ensureTriangleFanIB(int vertCount)` (file-static or namespace function):
- If `vertCount > 65535`: `DEBUG_FATAL` in debug; `DEBUG_REPORT_LOG_PRINT` + skip in release. Return false.
- If `ms_triangleFanIB && vertCount <= ms_triangleFanIBCapacityVerts`: return true (cache hit).
- Otherwise rebuild:
  - Determine new capacity = `max(kTriangleFanIBInitialCapacityVerts, max(ms_triangleFanIBCapacityVerts, vertCount))`; then double-up until `>= vertCount` AND clamp to 65535.
  - Allocate CPU staging array sized `(newCapacity - 2) * 3` `uint16_t` indices (~393K entries at 65535 cap = ~790KB; one-shot at install / very rare rebuild).
  - Fill with pattern `(0, i, i+1)` for `i = 1..newCapacity-2` â†’ `3*(newCapacity-2)` indices.
  - `CreateBuffer` with `USAGE_IMMUTABLE`, `BIND_INDEX_BUFFER`, `DXGI_FORMAT_R16_UINT`, `D3D11_SUBRESOURCE_DATA` pointing at the staging array.
  - On `FAILED(hr)`: clear `ms_triangleFanIB`, log, return false (next draw will retry).
  - On success: assign to `ms_triangleFanIB` ComPtr (auto-releases previous); update `ms_triangleFanIBCapacityVerts`; increment `ms_triangleFanIBRebuilds` counter; update `ms_triangleFanMaxVertices = max(ms_triangleFanMaxVertices, vertCount)`.
- Free CPU staging array on scope exit.

**1c.** `install()` hook: call `ensureTriangleFanIB(kTriangleFanIBInitialCapacityVerts)` to pre-allocate at initial capacity (avoids first-draw stall during world load). `remove()` hook: clear `ms_triangleFanIB` ComPtr (auto-managed but explicit for symmetry with other caches).

### Task 2 â€” drawTriangleFan + drawPartialTriangleFan reimplementation

```cpp
void Direct3d11_StateCache::drawTriangleFan()
{
    int const vertCount = ms_currentVBVertexCount;
    if (vertCount < 3)
        return;                       // need >= 3 verts for >= 1 triangle
    int const triCount = vertCount - 2;

    ++ms_drawTriangleFanCalls;

    if (!ensureTriangleFanIB(vertCount))
        return;                       // create failure or > 65535 -> skip

    if (!applyPreDrawState(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST))

        ms_triangleFanIB.Get(),
        DXGI_FORMAT_R16_UINT,
        0);

    Direct3d11_Device::getContext()->DrawIndexed(
        static_cast<UINT>(triCount * 3),
        0,
        0);
}
```

**2b.** Replace `drawPartialTriangleFan(startVertex, primitiveCount)` (currently StateCache.cpp:1446-1451):
```cpp
void Direct3d11_StateCache::drawPartialTriangleFan(int startVertex, int primitiveCount)
{
    if (primitiveCount < 1)
        return;
    int const vertCount = primitiveCount + 2;

    ++ms_drawPartialTriangleFanCalls;

    if (!ensureTriangleFanIB(vertCount))
        return;
    if (!applyPreDrawState(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST))

        static_cast<UINT>(primitiveCount * 3),
        0,
        static_cast<INT>(startVertex));
}
```
