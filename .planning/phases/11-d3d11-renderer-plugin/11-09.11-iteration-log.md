### Hypothesis

Same shape as Plan 11-09.10: STUB macro at Direct3d11.cpp:594 routes to `scaffold_fatal_stub`; the correct fix is real no-op `_impl` functions wired to slots. Three slots; all signature `void X(WCHAR const * name)`. No visual side-effect because PIX markers are debugger-side instrumentation only — silent in BOTH D3D9 and D3D11 when no PIX session is attached.

Engine-side call-site audit (from Plan 11-09.10 close investigation):
- `Graphics::pixSetMarker` callers: 2 — `Game.cpp:1009` ("Quit") + `GroundScene.cpp:2217` ("CloseLoadingScreen")
- `Graphics::pixBeginEvent` callers: 0
- `Graphics::pixEndEvent` callers: 0

We still no-op all 3 for symmetry + to prevent a similar surprise if future engine code adds pixBeginEvent / pixEndEvent callers.

### Investigation

Confirmed Gl_dll.def:226-228 signatures: `void (*pixSetMarker)(WCHAR const *)`, `void (*pixBeginEvent)(WCHAR const *)`, `void (*pixEndEvent)(WCHAR const *)`. Identical shape across all 3 slots.

Confirmed Plan 11-09.10's `_impl` precedent block at Direct3d11.cpp:~256 (file-scope inside `namespace Direct3d11Namespace`, alongside setBrightnessContrastGamma_impl / update_impl). Reused the same shape directly below the point-sprite block.

Confirmed STUB locations at Direct3d11.cpp:781-783 (line drift from 755-757 cited in the Plan 11-09.10 close summary — additional intermediate `STUB(setMouseCursor)` / `STUB(showMouseCursor)` / draw-dispatch wiring landed between the point-sprite block and the pix block in earlier plans).

### Root cause

Confirms hypothesis verbatim. STUB macro routes to scaffold_fatal_stub (FATAL); the correct behavior is silent no-op (PIX-for-Windows is silent without an attached PIX session in both D3D9 and D3D11). Engine `#if PRODUCTION == 0` guards mean these calls only fire in Debug builds, but they DO fire — the first one is GroundScene's `pixSetMarker(L"CloseLoadingScreen")` at the loading-screen-close transition.

### Fix

Per `11-09.11-PLAN.md` Tasks 1-2:

- **Task 2a:** 3 new file-scope `_impl` functions added to `namespace Direct3d11Namespace` in Direct3d11.cpp immediately after the Plan 11-09.10 point-sprite block. Empty bodies; WCHAR* param elided via `/*comment*/`. Preceded by a multi-paragraph comment block explaining the no-op rationale (PIX-for-Windows debugger-side instrumentation; PRODUCTION==0 guards), the engine-side call-site audit (2 pixSetMarker callers, 0 each for the others), and the Plan 11-12+ TODO for native ID3DUserDefinedAnnotation wire-up.
- **Task 2b:** Replaced the 3 STUB(pix*) lines at Direct3d11.cpp:781-783 with `ms_glApi.pix... = pix..._impl;` assignments + a short cross-reference comment pointing at the _impl block above.

D-05 preserved: only `Direct3d11.cpp` touched in the D3D11 plugin; the unstaged D3D9.cpp diff is Koogie's pre-existing 135-line working-tree patch (untouched).
