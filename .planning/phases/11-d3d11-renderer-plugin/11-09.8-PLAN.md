<objective>
Resolve the SM4 input-signature mismatch cascade-error surfaced by Plan 11-09.7's smoke -- 20+ `CreateInputLayout` ERRORs blocking the majority of post-login draws. Implement CODEX-endorsed reflection-driven input-layout assembly: reflect each VS bytecode's declared input signature, augment the layout with phantom zero-buffer elements at slot 15 for any non-system semantics the engine's VBFormat doesn't provide.

This is the C-lite / A2 hybrid from CODEX's Q2 ranking. NOT a shader-model rollback (Plan 11-09.6 vs_4_0 stays). NOT HLSL stripping (`Direct3d11_HlslRewrite.cpp` is already fragile). NOT hardcoded TEXCOORD2 — the fix is general.

Architecture:
- **Reflection-once-per-compile** in `Direct3d11_VertexShaderData.cpp:97` after successful D3DCompile. `D3DReflect` + `ID3D11ShaderReflection::GetInputParameterDesc` enumeration. Store compact filtered list (skip `SV_*` system-value inputs).
- **Phantom zero buffer** at `Direct3d11_Device::install` -- 16-byte immutable VB filled with zeros, BIND_VERTEX_BUFFER, owned by ComPtr through device lifetime. Accessor `getPhantomZeroBuffer()`.
- **Augment layout** during `Direct3d11_VertexDeclarationMap::getOrCreate` + `getOrCreateMultiStream` -- after VBFormat-driven element assembly, call new `Direct3d11_VertexBufferDescriptorMap::augmentWithPhantomElements(reflectedInputs, elements, count)` which appends missing semantics at InputSlot=15.
- **Bind phantom slot** in `Direct3d11_StateCache::applyPreDrawState` -- bind the phantom buffer at slot 15 with stride=0 / offset=0 unconditionally before every draw. Cheaper than gating per-VS; D3D11 is fine with unused-bound slots.

Per CODEX Q5: c0000005 is NOT assumed to share root cause with TEXCOORD2 ERRORs. Fix layouts first; if crash remains, Iter-2 with a real stack + minidump capture.

Scope ceiling: 1 iteration. If the c0000005 access violation remains after layout fix, that's Iter-2 territory.

Full CODEX consult preserved at `.planning/phases/11-d3d11-renderer-plugin/11-09.8-CODEX-CONSULT-texcoord2-signature-mismatch.md`.

### Task 1 — VS reflection capture

**1a.** Add `D3DReflect` call to `Direct3d11_VertexShaderData::compile` immediately after the successful D3DCompile path. Use the bytecode blob's GetBufferPointer / GetBufferSize. Get `ID3D11ShaderReflection*` via `IID_PPV_ARGS`.

**1b.** Enumerate inputs via `ID3D11ShaderReflectionConstantBuffer::GetDesc` → `D3D11_SHADER_DESC.InputParameters`. For each `i in [0, InputParameters)`: call `GetInputParameterDesc(i, &desc)`. Filter out `desc.SystemValueType != D3D_NAME_UNDEFINED` (skip SV_VertexID, SV_InstanceID, etc.).

**1c.** Define a compact struct (e.g., `ReflectedInput`) holding the bare minimum: `char SemanticName[16]`, `UINT SemanticIndex`, `UINT ComponentMask`, `D3D_REGISTER_COMPONENT_TYPE ComponentType`. Store as `std::vector<ReflectedInput>` member in `Direct3d11_VertexShaderData`.

**1d.** Add public accessor `getReflectedInputs() const -> std::vector<ReflectedInput> const &`.

**1e.** Reflection is a one-time post-compile cost. The disk-cache path (`tryLoad`) ALSO needs to re-reflect from cached bytecode at load time (cached blobs don't carry reflected metadata separately). This is OK — reflect on EVERY successful compile + on EVERY successful tryLoad.

### Task 2 — Phantom zero buffer

**2b.** In `Direct3d11_Device::create` (after device creation succeeds, before any draw work), create a 16-byte zero-initialized ID3D11Buffer:

### Task 3 — Layout augmentation

**3a.** Add `augmentWithPhantomElements()` to `Direct3d11_VertexBufferDescriptorMap.h` declaration. Signature:
```cpp
// Plan 11-09.8: append phantom elements for VS-declared inputs not
// covered by the format-derived elements. Phantom elements use
// InputSlot=15 with AlignedByteOffset=0 (zero-stride read).
//
// Returns the new total count (which may equal the input count if
// no augmentation was needed) or a negative value on capacity
// overflow.
static int augmentWithPhantomElements(
    std::vector<Direct3d11_VertexShaderData::ReflectedInput> const &reflectedInputs,
    D3D11_INPUT_ELEMENT_DESC *outDesc,
    int currentCount,
    int maxElements);
```

**3b.** Implement in `Direct3d11_VertexBufferDescriptorMap.cpp`:
- Iterate reflected inputs.
- For each non-SV reflected input, scan the existing `outDesc[0..currentCount)` for a matching (SemanticName, SemanticIndex) pair.
- If not found, append a phantom element at `outDesc[currentCount]`: SemanticName + SemanticIndex from reflected input; Format derived from reflected ComponentType + ComponentMask (use existing reflectedToDxgi() helper or inline it); InputSlot=15; AlignedByteOffset=0; InputSlotClass=D3D11_INPUT_PER_VERTEX_DATA; InstanceDataStepRate=0.
- Increment currentCount; return new total or -1 on overflow.

**3c.** `reflectedToDxgi()` helper -- maps (ComponentType, ComponentMask) to DXGI_FORMAT:
- ComponentMask 0x1 → R (scalar), 0x3 → RG, 0x7 → RGB, 0xF → RGBA.
- ComponentType float (3) → R32_FLOAT / R32G32_FLOAT / R32G32B32_FLOAT / R32G32B32A32_FLOAT.
- ComponentType uint (1) → R32_UINT / R32G32_UINT / R32G32B32_UINT / R32G32B32A32_UINT.
- ComponentType sint (2) → R32_SINT / R32G32_SINT / R32G32B32_SINT / R32G32B32A32_SINT.
- Default: float4 (DXGI_FORMAT_R32G32B32A32_FLOAT). Phantom zero-input is type-agnostic for our purposes.

<risks>
1. **Reflection cost on tryLoad** — disk-cache hit path needs to re-reflect from cached bytecode. Negligible per-VS (microseconds typical), no caching of reflection results across runs. Acceptable.

2. **Phantom buffer interference with multi-stream slots** — Plan 11-09.7 wires multi-stream up to slot 1. Slot 15 is well clear. Risk: zero.

3. **DXGI_FORMAT derivation accuracy** — the reflectedToDxgi() helper maps (ComponentType, ComponentMask) to DXGI format. For phantom slots reading zero, the format type doesn't matter functionally. But the format MUST be byte-size-compatible with stride=0 reading; D3D11 validates this. Mitigation: choose largest type per mask width (always emit R32_*..R32G32B32A32_*) — covers all cases safely.

4. **D3DReflect availability** — already linked via Plan 11-05 (d3dcompiler_47). Confirm `d3d11shader.h` + `d3dcompiler.h` are accessible from Direct3d11_VertexShaderData translation unit.

5. **Engine-side draws via direct IASetVertexBuffers** — engine doesn't bypass our StateCache, but worth a grep. If the engine ever directly clobbers slot 15, our phantom buffer is overwritten. Mitigation: rebind in applyPreDrawState (which we are doing).

6. **D3D11 debug-layer: extra binds may trigger warnings?** D3D11 typically warns on "buffer bound but not consumed by VS." Slot 15 will be bound + consumed (the phantom semantics ARE referenced by the augmented layout). No warning expected. If a warning surfaces: silenceable via D3D11_INFO_QUEUE_FILTER (Plan 11-08 InfoQueue infrastructure already in place).

7. **D-05 invariant** — D3D11 plugin only. No engine-side edits expected in this plan. Risk: zero.

8. **BSOD** — pure CPU-side input-layout work + per-draw vertex-buffer binding. No new GPU resource allocations beyond the one 16-byte phantom buffer (install-time only). Risk: zero.
</risks>
