The InstallTimer log marker (verified at src/engine/shared/library/sharedDebug/src/shared/InstallTimer.cpp:34):
- Config key activating it: `[SharedDebug/InstallTimer] enabled=true`
- Output format: `InstallTimer:%*c%6.4f %d %s` (indent + elapsed + bytes + description)
- Phase 9 completion marker: a line containing "Game::install" inside the Logs file
- The line appears at indent depth 1 (immediate child of rootInstallTimer)

LoginConnection wire (verified at src/engine/client/library/clientGame/src/shared/network/LoginConnection.cpp:53-79):
- On UDP open, sends LoginClientId(username, password, NetworkVersionId="20100225-17:43")
- NetworkVersionId is BYTE-IDENTICAL to swg-main's value (verified by Plan 5 + RESEARCH.md)
- swg-main validateClient() with externalAuthURL="" accepts any credentials -> authOK=1
- Server replies with LoginEnumCluster (cluster list) + EnumerateCharacterId (per-cluster character roster)

Boot phase markers expected in the log (from docs/recon/05-client-boot-sequence.md):
- Phase 3 reached: "SetupSharedFoundation::install" InstallTimer line
- Phase 5 reached: "SetupSharedFile" or TreeFile open lines
- Phase 7 reached: "SetupClientGraphics::install"
- Phase 8 reached: "Game::install" entry (start)
- Phase 9 reached: "Game::install" exit InstallTimer line (MVB-1 marker)

Triage runbook categories (from RESEARCH.md "Boot Failure Triage Runbook" section):
1. Missing runtime DLL — Windows modal "code execution cannot proceed" + no log file
2. TRE archive not found — "TreeFile: warning, opening 'sku0_client.toc' failed" + black window
3. D3D9 device init failure — boot reaches SetupClientGraphics, then exits without window
4. Config key not found — "Config file not specified" or "Connection lost" on submit
