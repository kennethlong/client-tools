Glossary

SWG
Star Wars Galaxies. Sandbox MMO by Sony Online Entertainment, launched 2003, sunset 2011.
SOE
Sony Online Entertainment, the developer/publisher (later renamed Daybreak).
NGE
"New Game Enhancements", the November 2005 redesign of SWG. The codebase in this repo is the late-NGE era. Most "NGE" emulators target this design.
CU
"Combat Upgrade", an April 2005 redesign that came between launch and NGE.
Pre-CU
The original 2003 design before either redesign — what SWGEmu chose to recreate.
JTL
Jump to Lightspeed, the 2004 space-flight expansion (sku.1).
RotW
Rage of the Wookiees, the Kashyyyk expansion (sku.2).
ToOW
Trials of Obi-Wan, the Mustafar adventure pack (sku.3).
sku
Stock-keeping unit — used here as the directory split between base content and expansions in dsrc/sku.0/, sku.1/, sku.2/, sku.3/, sku.ja/ (Japanese release).
dsrc
"Data source" — the authoring side of game content (scripts, datatables, templates) before it gets compiled and packed.
data
The compiled output of dsrc — what the runtime reads. Not in the repo (only the manifests describing it are).
IFF
Interchange File Format. Chunked binary format used for nearly every authored asset (meshes, datatables, templates, world snapshots). Originating from the Electronic Arts EA-IFF spec.
TRE
SWG's archive format. A .tre bundles many .iff files plus textures, sounds, etc., with an index and per-entry MD5.
STF
String Table Format — localisation entries.
tab
Tab-separated spreadsheet with a typed header row. The authoring form of a datatable; compiled to an .iff at build time.
tpf
Template source — declarative description of an object template, compiled to .iff by TemplateCompiler.
script / scriptlib
SWG's scripting language. .script = handlers for one object kind. .scriptlib = shared library code. .java next to them is the SWG dialect, not vanilla Java.
scriptbin
Compiled script bytecode loaded by the JVM embedded in SwgGameServer.
WS / world snapshot
Static-object placement data for a planet shard. Edited with TerrainEditor / WorldSnapshotViewer.
God client
The in-game admin/edit client (SwgGodClient). Designers used it to place objects in the world live.
obj_id
The 64-bit network identifier for any game object. Persistent across sessions.
TaskManager
Per-host process supervisor. Reads _taskmanager.cfg/.rc and spawns/respawns the cluster's processes locally.
StationKey
The session token from Sony's Station authentication. Validated by LoginServer in production. Set validateStationKey=0 to bypass on private servers.
OCI
Oracle Call Interface — the C library used by the C++ servers to talk to Oracle. databaseProtocol=OCI appears in every server cfg.
DSN
Data Source Name — the Oracle TNS alias the server connects to.
SWGEmu
The longest-running independent SWG emulator. Clean-room C++ codebase targeting Pre-CU. Not based on this leak.
ProjectSWG / NGECore
An NGE-era emulator implemented in Java. Independent of the C++ codebase here.
SWG-Source / StellaBellum
The community fork that is built on the leaked code (this repo's descendant). StellaBellum is their dev VM and an active server.
whitengold
The name of this archival repository on GitHub (AconiteX/whitengold) — preserved as the unmodified Feb 28, 2015 snapshot. The name is just the repo name; it doesn't denote a particular branch within SOE's depot. The Perforce paths in publish/source_revisions.txt show this came from the //depot/swg/test/ branch.
SOE Station / soePlatform / stationapi
SOE's internal account management, friend list, and entitlement platform — gone since Sony divested SOE in 2015.
Vivox
Third-party voice chat SDK. SWGVoiceService.exe is the local helper.
DPVS
Database of Potentially Visible Sets — Umbra's occlusion culling library. Used by clientGraphics.
Bink / Miles
RAD Game Tools' video and audio middleware. Bink for cinematics, Miles for in-game audio (Mss32.dll).
Alienbrain
NxN's asset-management system used in SOE's content pipeline. The XDK headers are here for editor integration.
