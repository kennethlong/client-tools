Overview

This repository (whitengold) is a backup of the February 28, 2015 leak of the Star Wars Galaxies source code. The leak originated from a former SOE employee in 2013 and represents the ~2010 production codebase used to operate the live game until shutdown in 2011. The README inside the repo confirms this provenance.

It is not a finished, runnable build. It is one piece of a sprawling multi-process system that depended on assets, an Oracle database schema, Sony's authentication infrastructure, and a Linux/MSVC toolchain that no longer exists in working form. The community-maintained continuation of this tree is SWG-Source — that is where practical "run a server" work happens today.

How to use these docs: start with Architecture to see how the pieces fit, then visit Servers and Client for the C++ side, Scripting for the gameplay rules, and finally Gaps + Run Locally to understand what stops you from booting it.

src/configure.in, src/bootstrap, src/Makefile.amGNU autotools entry point for the Linux server build (see src/BUILDING).

Two big halves: a heavyweight DirectX 9 client talking custom UDP to a fleet of cooperating Linux servers. Gameplay rules live in Java-derived scripts under dsrc/, hosted by an embedded JVM inside SwgGameServer. Persistence is Oracle. Authentication was Sony's Station service.
