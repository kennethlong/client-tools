Build-toolchain expectations

The repo's src/BUILDING file lists what was on the dev workstations:

  • automake >= 1.6 < 1.7
  • autoconf >= 2.57
  • libtool-1.4x
  • gcc-2.95-3 or gcc-3.2.x
  • IBM Java 1.3.1 in /usr/java
  • STLPort 4.5.3 sources
  • Oracle 9.2.0 in /opt/app/oracle/product/9.2.0
Most of those are end-of-life. A from-scratch build of the unmodified tree on a current Linux distro will not succeed without porting. Community efforts (SWG-Source) maintain a patched fork that does build on more recent toolchains, paired with an Oracle 12c instead of 9i.

Windows build expects

  • Visual Studio 2005 (swg.sln is format 9.00).
  • DirectX 9 SDK (early 2005-era headers).
  • Maya SDK for the relevant version (4 / 5 / 7 / 8).
  • Oracle Instant Client / OCI for Windows.
  • The MSVC 7.1 runtime (msvcr71.dll) — already in exe/win32/.
